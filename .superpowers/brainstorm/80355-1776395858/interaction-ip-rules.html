双轨混合型 · 交互规则与 IP 保护

目标：让用户“先相信、再试用、后导入”，同时保护作者的源码和核心逻辑

用户能看到什么
公开展示
  • 策略名称、作者、分类、标签
  • 区间收益率、最大回撤、胜率、手续费等摘要指标
  • 策略简介、适用市场、风险提示
  • 作者帖子/复盘内容
默认不公开
  • 源码全文
  • 触发条件细节
  • 完整参数逻辑推导
  • 可导出的原始策略文件
用户能做什么
试用回测

可以改公开参数、选时间区间、查看结果，但看不到源码与私密逻辑。

一键导入

导入后进入个人策略库，生成 sealed 副本，可继续回测和托管。

社区互动

点赞、评论、收藏、转发帖子，但不会获得策略源码所有权。

策略详情页主操作区建议

主按钮顺序

顺序表达的是：先验证效果，再决定导入，社区讨论作为辅助决策信息。

导入后状态

  • 已导入：按钮变为“已在策略库中”
  • 仍可继续试用回测
  • 点击进入我的策略库副本详情
知识产权保护规则
规则 1：帖子只挂卡片，不承载完整策略资产

帖子是传播内容，不应该变成绕过保护机制的“源码展示页”。

规则 2：试用回测只开放白名单参数

用户可以体验，不代表能无限拆解策略。

规则 3：导入得到的是运行权，不是文件所有权

UI 和文案都应明确“已解锁使用权”，避免用户误以为自己拿到了源码。

A

规则对

继续讲 MVP 功能边界，避免第一版做得过重。

B

规则要调

某些可见信息、可调参数或导入后的权限还需要改。

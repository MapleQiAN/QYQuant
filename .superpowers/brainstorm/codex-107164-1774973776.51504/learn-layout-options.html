量化课程页改版方向

目标：消除宽屏右侧空白，同时承载更专业、更密集的量化课程内容与参数词典。

A

Research Terminal

推荐。全宽流体布局，左侧课程索引，右侧主内容，章节内部支持 2-3 列知识面板与参数表。

B

Notebook Docs

文档感更强，阅读舒适，但宽屏利用率较弱，容易再次出现留白感。

C

Dashboard Atlas

视觉冲击强，适合信息总览，但不如 A 适合长课程与术语解释的连续学习。

我推荐 A 的原因

  • 宽屏可以真正吃满，不会右侧空一大片。
  • 更适合“课程 + 术语 + 参数表 + AI 模块”同时出现。
  • 后续继续扩章节时，结构最稳，不容易重做。

需要注意

  • 内容密度更高，必须靠清晰层级和折叠节奏控制阅读负担。
  • 移动端需要降成单列，不然会显得过挤。

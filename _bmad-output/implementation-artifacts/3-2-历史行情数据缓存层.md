# Story 3.2: 历史行情数据缓存层

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

作为平台，
我希望建立 PostgreSQL 时序缓存层，优先从本地缓存获取历史数据，
以便降低外部 API 调用量并在数据源不可用时仍能服务回测。

## 验收标准

1. **market_data_cache 表已创建**
   - Given market_data_cache 表已创建（symbol VARCHAR(20)、trade_date DATE、open/high/low/close NUMERIC、volume BIGINT、source VARCHAR(20) DEFAULT 'joinquant'、cached_at TIMESTAMPTZ，PRIMARY KEY (symbol, trade_date)）
   - When 回测任务请求某 symbol 的历史数据
   - Then 优先查询缓存，命中率目标 ≥90%（NFR7）

2. **缓存未命中时调用外部 API**
   - And 缓存未命中时调用聚宽 JQData API，结果写入缓存后返回

3. **数据源不可用时的降级**
   - Given 聚宽 API 不可用
   - When 回测任务请求数据
   - Then ≤3 秒内返回已缓存的数据，并在响应中附加"数据范围提示"（NFR9）
   - And 不出现空白页面或未捕获异常

## Dev Notes

### 架构模式与约束

- **技术栈**：Python 3.11 + Flask + SQLAlchemy + PostgreSQL
- **缓存策略**：不使用 Redis 缓存行情数据（体积大，冷数据多，PG 已足够）
- **外部 API**：聚宽 JQData（`pip install jqdatasdk`），需要注册账号获取 API 凭证
- **数据格式**：OHLCV（Open/High/Low/Close/Volume），日线级别
- **数据量预估**：单标的10年日线数据约 2500 行，100 个标的约 25 万行，PG 完全够用

### 现有代码分析（关键！）

**⚠️ 需要新建模块，但要与现有代码模式对齐：**

1. **`backend/app/models.py`**：所有模型在单文件中，使用 `gen_id()` 生成 UUID，时间使用 `now_ms` 毫秒时间戳
   - **注意**：架构规范要求新表使用 TIMESTAMPTZ 类型，与现有的 BigInteger 时间戳不同
   - MarketDataCache 应遵循架构规范使用 TIMESTAMPTZ

2. **`backend/app/backtest/engine.py`**：现有回测引擎直接执行回测
   - 数据获取逻辑可能在此文件或 `providers.py` 中
   - 缓存层需要作为数据获取的中间层

3. **`backend/app/backtest/providers.py`**：可能包含现有数据提供者
   - 新的缓存服务应替换或包装现有数据提供者

### 关键技术细节

**MarketDataCache 表设计要点：**
```sql
CREATE TABLE market_data_cache (
  symbol VARCHAR(20) NOT NULL,
  trade_date DATE NOT NULL,
  open NUMERIC NOT NULL,
  high NUMERIC NOT NULL,
  low NUMERIC NOT NULL,
  close NUMERIC NOT NULL,
  volume BIGINT NOT NULL,
  source VARCHAR(20) DEFAULT 'joinquant',
  cached_at TIMESTAMPTZ DEFAULT NOW(),
  PRIMARY KEY (symbol, trade_date)
);
```

**缓存查询逻辑伪代码：**
```python
def get_market_data(symbol, start_date, end_date):
    # 1. 查询缓存
    cached = MarketDataCache.query.filter(
        MarketDataCache.symbol == symbol,
        MarketDataCache.trade_date.between(start_date, end_date)
    ).all()

    # 2. 计算缺失日期
    missing_dates = compute_missing(start_date, end_date, cached)

    # 3. 如有缺失，调用外部 API
    if missing_dates:
        try:
            new_data = joinquant.fetch_daily_data(symbol, min(missing_dates), max(missing_dates))
            bulk_insert(new_data)
            cached.extend(new_data)
        except ExternalAPIError:
            # 降级：返回已缓存数据 + 范围提示
            return cached, {"notice": f"数据仅覆盖已缓存范围"}

    return cached, None
```

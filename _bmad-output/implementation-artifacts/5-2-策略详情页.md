# Story 5.2: 策略详情页

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

作为用户，
我希望查看策略详情页，包含完整描述、展示指标和 Equity Curve，
以便在不看代码的前提下判断策略是否值得试用。

## Acceptance Criteria

1. **Given** 用户点击任意策略卡片
   **When** 访问 `GET /api/v1/marketplace/strategies/:id`
   **Then** 详情页展示：名称、作者昵称/头像、分类、描述、标签、发布时间（北京时间 ISO 8601+08:00）（FR26）
   **And** 展示发布者设置的展示指标（`display_metrics` JSONB，FR32）
   **And** 展示策略 Equity Curve 图（来自 `result_storage_key`）
   **And** 若 `is_verified=true` 显示蓝色"平台验证回测"标识（FR27）
   **And** 策略代码对访问者完全不可见，无任何下载入口（FR34）

## Dev Notes

### 架构约束与模式

- **前端框架**: Vue 3 + Composition API + TypeScript + Vite + Pinia
- **后端框架**: Flask + 蓝图，路由前缀 `/api/v1/`
- **数据库**: PostgreSQL + Alembic
- **时区规范**: 数据库 UTC，API 输出 `+08:00`
- **API 响应格式**: `{ "data": {...}, "meta": {...} }`

### UX 设计要求

- **详情页布局**: 参考极简金融仪表盘风格
- **Equity Curve 图表**: 复用 Epic 3 的 `EquityCurveChart` 组件（如已存在）
- **核心三指标大字展示**: 32px/700wt 字号（累计收益率、最大回撤、夏普比率）
- **验证徽章**: 复用 Story 5.1 的 `VerifiedBadge.vue`
- **CTA 按钮**: 黑色 `#1A1A1A` 填充，白字，8px 圆角
- **时间显示**: 北京时间，格式 ISO 8601+08:00
- **标签展示**: 胶囊形 chip，灰色边框

### 安全要求（FR34 — 代码不可见）

- **API 层**: 详情接口**绝不返回** `code_encrypted`、`code_hash`、`storage_key`（storage_key 仅用于 equity curve 数据读取，不直接暴露给前端）
- **前端层**: 无任何"查看代码"、"下载"入口
- **Equity Curve 数据**: 通过独立接口返回时间序列数据，不暴露完整回测结果文件路径

### 关键依赖

- **跨 Epic 依赖**: Epic 3 的 EquityCurveChart 组件（如已实现则复用，否则需自行实现简版）

### 防踩坑提醒

- **不要** 在任何 API 响应中返回 `code_encrypted` 或 `code_hash` — 这是安全红线
- **不要** 重新实现 Equity Curve 图表 — 检查 Epic 3 是否已有组件可复用
- **不要** 在前端直接拼接 `result_storage_key` URL — 通过后端代理接口获取数据
- **不要** 使用 `datetime.now()` — 使用 UTC 存储 + 前端格式化为北京时间
- **display_metrics** 是 JSONB，字段名不固定，需动态渲染

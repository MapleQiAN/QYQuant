# Story 6.3: 评论帖子

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

作为已登录用户，
我希望在帖子下方发表评论，
以便与帖子作者和其他用户交流讨论。

## Acceptance Criteria

1. **Given** 用户在帖子详情页评论框输入内容并提交
   **When** 调用 `POST /api/v1/posts/:id/comments`（body: `{"content": "..."}`)
   **Then** `post_comments` 表（见 Task 1）创建新记录，`posts.comments_count +1`（FR37）
   **And** 评论内容长度限制 ≤500 字符，超出返回 `422` 并说明限制
   **And** 返回格式 `{"id": "...", "content": "...", "user_id": "...", "created_at": "<北京时间ISO>", "author": {"nickname": "...", "avatar_url": "..."}}`

2. **Given** 新评论提交成功
   **When** 前端收到响应
   **Then** 新评论立即出现在评论列表底部（无需刷新页面）
   **And** 帖子卡片上的评论数立即 +1（乐观更新）

3. **Given** 访问帖子详情页
   **When** 调用 `GET /api/v1/posts/:id/comments`
   **Then** 返回评论列表，按 `created_at` 正序排列，支持分页（每页 20 条）
   **And** 返回格式 `{"items": [...], "total": N, "page": 1, "per_page": 20}`
   **And** 每条评论含作者信息（nickname、avatar_url）

4. **Given** 请求的 post_id 不存在
   **When** 调用任意评论端点
   **Then** 返回 `404` 错误

5. **Given** 未认证用户
   **When** 调用 `POST /api/v1/posts/:id/comments`（写评论）
   **Then** 返回 `401` 错误
   **And** `GET /api/v1/posts/:id/comments`（读评论）不需要认证

    ```python
    def get_comments(post_id):
        post = Post.query.get(post_id)
        if not post:
            return error_response(404, 'post_not_found', '帖子不存在', 404)

        page = request.args.get('page', 1, type=int)
        per_page = min(request.args.get('per_page', 20, type=int), 50)

        pagination = (
            PostComment.query
            .filter_by(post_id=post_id)
            .order_by(PostComment.created_at.asc())
            .paginate(page=page, per_page=per_page, error_out=False)
        )

        items = []
        for c in pagination.items:
            author = User.query.get(c.user_id)
            items.append({
                'id': c.id,
                'content': c.content,
                'user_id': c.user_id,
                'created_at': format_beijing_iso(c.created_at),
                'author': {
                    'nickname': author.nickname if author else '',
                    'avatar_url': author.avatar_url if author else ''
                }
            })

        return ok({
            'items': items,
            'total': pagination.total,
            'page': page,
            'per_page': per_page
        })
    ```

### 架构约束与模式

### ⚠️ 关键陷阱：使用新表 post_comments 而非旧表 comments

**已存在的旧 `comments` 表（forum.py 使用）：**
```sql
comments (id, post_id, author TEXT, avatar TEXT, content TEXT,
          timestamp BIGINT, user_id FK)
```

**现有 `Comment` 模型（models.py）也映射到此表，`forum.py` 引用它。**

**本 Story 的正确做法：**
- 创建全新的 `post_comments` 表（不与旧 `comments` 表冲突）
- 在 `models.py` 中新增 `PostComment` 类（`__tablename__ = 'post_comments'`）
- **绝对不要修改**现有 `Comment` 类

这与 Story 6.1（新增 `community.py` 代替 `forum.py`）和 Story 6.2（新增 `post_interactions` 代替 `likes`/`favorites`）的模式一致。

### 前端乐观更新模式

```typescript
// useCommunityStore.ts
async createComment(postId: string, content: string) {
  // 1. 乐观追加（临时 ID）
  const tempComment: CommentItem = {
    id: `temp-${Date.now()}`,
    content,
    user_id: authStore.user.id,
    created_at: new Date().toISOString(),
    author: { nickname: authStore.user.nickname, avatar_url: authStore.user.avatar_url }
  }
  if (!this.commentsByPostId[postId]) this.commentsByPostId[postId] = []
  this.commentsByPostId[postId].push(tempComment)

  // 2. 乐观更新帖子评论数
  const post = this.posts.find(p => p.id === postId)
  if (post) post.comments_count++

  try {
    const real = await communityApi.createComment(postId, { content })
    // 3. 替换临时评论为真实数据
    const idx = this.commentsByPostId[postId].findIndex(c => c.id === tempComment.id)
    if (idx !== -1) this.commentsByPostId[postId][idx] = real
  } catch (e) {
    // 4. 回滚
    this.commentsByPostId[postId] = this.commentsByPostId[postId].filter(
      c => c.id !== tempComment.id
    )
    if (post) post.comments_count--
    throw e
  }
}
```

# Story 2.4: 账号注销与数据删除

Status: done

## Story

作为已登录用户，
我希望注销账号并删除所有个人数据，
以便行使《个人信息保护法》赋予的被遗忘权。

## Acceptance Criteria (验收标准)

1. **二次验证**：用户在账户设置页点击"注销账号"，需输入手机验证码确认（复用 send-code 端点）
2. **软删除用户**：调用 `DELETE /api/v1/users/me`，验证通过后软删除 users 记录（写入 deleted_at，不物理删除）
3. **数据脱敏**：用户昵称替换为"已注销用户"，手机号字段清空
4. **强制登出所有设备**：所有 Refresh Token 立即写入 Redis 黑名单
5. **审计日志**：操作写入 audit_logs 表（operator_id、action="user_delete"）
6. **已发布策略处理**：本 Story 仅标记用户状态，已发布策略的处理逻辑由 Epic 10 Story 10.x 实现

### 软删除策略

**注销后的用户记录状态：**
```sql
UPDATE users SET
  phone = NULL,
  nickname = '已注销用户',
  avatar_url = '',
  bio = '',
  deleted_at = NOW()
WHERE id = :user_id;
```

**关键决策：**
- **不物理删除**：保留 id 和 deleted_at 用于审计追溯和外键完整性
- **phone 设为 NULL**：允许该手机号重新注册新账户
- **其他关联数据**：策略、帖子等由各自的 Epic 处理（本 Story 仅处理 users 表）

### 审计日志工具函数

```python
# backend/app/utils/audit.py
def log_audit(operator_id: str, action: str, target_type: str,
              target_id: str, details: dict = None):
    """写入审计日志"""
    from ..models import AuditLog
    from ..extensions import db
    audit = AuditLog(
        operator_id=operator_id,
        action=action,
        target_type=target_type,
        target_id=target_id,
        details=details or {}
    )
    db.session.add(audit)
    # 不单独 commit，由调用方统一提交事务
```

### 全设备登出流程

1. 查询 `refresh_tokens WHERE user_id = :id AND revoked_at IS NULL`
2. 批量更新 `revoked_at = NOW()`
3. 对每个 Token，计算剩余有效期，写入 Redis 黑名单：
   ```
   token:blacklist:{jti} → "1"  TTL = max(0, expires_at - now)
   ```
4. 清除当前请求的 Cookie

### 合规要求 (NFR23)

- 《个人信息保护法》要求的"被遗忘权"：用户可请求删除所有个人数据
- 软删除 + 数据脱敏满足合规要求：个人可识别信息（手机号、昵称、头像）已清除
- 审计日志保留 ≥ 90 天（NFR29），仅 admin 可查看
- 注销操作不可撤销（需在前端 UI 有明确警告，由前端 Story 处理）

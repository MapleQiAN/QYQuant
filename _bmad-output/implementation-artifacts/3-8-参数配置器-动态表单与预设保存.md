# Story 3.8: 参数配置器（动态表单与预设保存）

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

作为已登录用户，
我希望在 Web 上通过动态生成的表单配置策略参数，并保存为预设，
以便无需修改代码即可调整参数进行多次回测。

## 验收标准

1. **动态表单渲染**
   - Given 策略的 strategy.json 中定义了 parameters 数组
   - When 用户访问策略参数配置页
   - Then 根据参数类型自动渲染控件：slider（含 min/max/step）、dropdown（枚举）、数字输入框（QYSP-FR8）

2. **参数说明展示**
   - And 参数说明以 tooltip 方式展示

3. **实时验证**
   - And 输入不合法时实时显示错误提示

4. **预设保存**
   - When 用户点击"保存预设"
   - Then 当前参数组合存入 strategy_parameter_presets 表，可供下次回测复用

## Dev Notes

### 架构模式与约束

- **数据架构决策**：新增 `strategy_parameter_presets` 表（非 JSONB 字段内嵌）
- **理由**：支持单策略多套预设（如"保守"/"激进"），多用户场景结构更清晰
- **前端**：Vue 3 + Pinia，组件复用现有 UI 框架
- **QYSP 对齐**：参数定义格式与 QYSP strategy.json 的 parameters 数组一致

### 关键技术细节

**参数类型到控件映射：**
| 参数类型 | 条件 | 控件 |
|---------|------|------|
| int/float | 有 min + max + step | Slider |
| enum | 有 options | Dropdown |
| int/float | 无范围 | NumberInput |
| string | — | TextInput |

**strategy_parameter_presets 表：**
```sql
CREATE TABLE strategy_parameter_presets (
  id VARCHAR PRIMARY KEY,
  strategy_id VARCHAR REFERENCES strategies(id),
  user_id VARCHAR REFERENCES users(id),
  name VARCHAR(100) NOT NULL,
  parameters JSONB NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX idx_presets_strategy_user ON strategy_parameter_presets(strategy_id, user_id);
```

# Story 6.1: 发布帖子

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

作为已登录用户，
我希望在策略广场发布帖子（可关联策略），
以便分享我的策略心得和回测经验。

## Acceptance Criteria

1. **Given** 用户点击"发布帖子"
   **When** 提交 `POST /api/v1/posts`（content、可选 strategy_id）
   **Then** posts 表创建新记录，返回帖子数据（含 created_at 北京时间，FR35）
   **And** 帖子内容长度限制 ≤2000 字符，超出返回 422 并说明限制
   **And** 若关联了 strategy_id，帖子详情页展示该策略的卡片预览
   **And** 发布成功后帖子立即出现在策略广场信息流中（`GET /api/v1/posts`）

## Dev Notes

### 架构约束与模式

- **前端框架**: Vue 3 + Composition API + TypeScript + Pinia
- **后端框架**: Flask + flask-smorest 蓝图，认证用 `@jwt_required()` + `get_jwt_identity()`
- **数据库**: PostgreSQL + Alembic 迁移，新 migration 文件放在 `backend/migrations/versions/`
- **API 路径**: 新 Community 功能使用 `/api/v1/posts/`（不是旧的 `/api/forum/posts`）
- **时区**: `created_at` 存储为 UTC（`DateTime(timezone=True)`），API 响应通过 `format_beijing_iso()` 转为北京时间（来自 `app.utils.time`）
- **响应格式**: 使用 `ok()` 封装数据（来自 `app.utils.response`）

### ⚠️ 关键陷阱：现有 Post 模型冲突

**现有 `models.py` 中 `Post` 模型的字段**：
```python
title = db.Column(db.String, nullable=False)
author = db.Column(db.String, nullable=False)   # 文字作者名，非 FK
avatar = db.Column(db.String, nullable=False, default='')
likes = db.Column(db.Integer, default=0)
comments = db.Column(db.Integer, default=0)
timestamp = db.Column(db.BigInteger, nullable=False)  # 毫秒时间戳
user_id = db.Column(db.String, db.ForeignKey('users.id'))
```

**架构目标字段**（来自 architecture.md 领域5）：
```sql
posts (
  id, user_id, strategy_id NULLABLE,
  content TEXT, likes_count INT DEFAULT 0,
  comments_count INT DEFAULT 0,
  created_at TIMESTAMPTZ
)
```

**迁移策略**：
- 在 Alembic migration 中只**新增**字段，不删除旧字段（防止回归）
- 新的 `community.py` 蓝图只使用新字段（content, strategy_id, likes_count, comments_count）
- 旧的 `forum.py` 蓝图保持不变，继续使用旧字段

### ⚠️ 关键陷阱：蓝图注册

现有 `__init__.py` 已注册 `forum_bp`（`/api/forum`）。新建的 `community_bp` 注册到 `/api/v1`：
```python
# backend/app/__init__.py 中追加
from .blueprints.community import bp as community_bp
# ...
app.register_blueprint(community_bp)
```

**不要**删除或修改现有 `forum_bp` 注册——会破坏现有功能。

### 关键依赖

- **前置依赖**: 无（Epic 6 首个 Story）
- **数据依赖**: `users` 表已存在（Epic 2 已完成）；`strategies` 表已存在（Epic 1）
- **跨 Story 依赖**: Story 6.2（点赞/收藏）会在 `post_interactions` 表中记录，本 Story 不创建该表

### 防踩坑提醒

- **不要**修改或删除 `forum.py` 蓝图及其相关代码——会破坏已有功能
- **不要**修改 `Post` 模型现有字段（`title`, `author`, `avatar` 等）——只新增字段
- **不要**在 `content` 字段上做 nullable=False 的数据库约束（旧数据没有 content）——在应用层验证必填
- **不要**直接使用 `timestamp` 字段存时间——新代码用 `created_at` (TIMESTAMPTZ)，然后用 `format_beijing_iso()` 返回北京时间
- **content 长度验证**：在应用代码中限制 `len(content) <= 2000`，**不要**仅依赖数据库约束

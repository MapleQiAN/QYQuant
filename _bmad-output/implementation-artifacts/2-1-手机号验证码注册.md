# Story 2.1: 手机号验证码注册

Status: done

## Story

作为新用户，
我希望通过手机号接收验证码并完成注册，
以便快速开始使用平台而无需记住额外的账号密码。

## Acceptance Criteria (验收标准)

1. **数据库表创建**：执行 migration 后，`users` 表按架构定义重构（phone 替换 email，新增 role/plan_level/is_banned/deleted_at 等字段），`refresh_tokens` 表和 `audit_logs` 表已创建（含所有字段和约束）
2. **发送验证码**：用户填写手机号点击"获取验证码"，`POST /api/v1/auth/send-code` 通过短信服务商发送 6 位验证码，验证码存入 Redis（TTL 5 分钟）
3. **频率限制**：同一手机号 60 秒内再次请求返回 429 错误，提示"请 X 秒后重试"
4. **注册+登录**：用户提交手机号+验证码+昵称，调用 `POST /api/v1/auth/login`，首次自动注册（创建 users 记录，role=user, plan_level=free），返回 Access Token（15 分钟有效期）+ HttpOnly Cookie 存储 Refresh Token（30 天）
5. **已注册用户登录**：已注册手机号提交验证码后直接登录，不重复创建用户

**现有代码模式（必须遵循）：**
- ID 生成：`str(uuid.uuid4())` — 所有模型主键用 String UUID
- 时间戳：`BigInteger` 毫秒级 (`now_ms()`) — 但架构要求用 `TIMESTAMPTZ`，本 Story 开始迁移到 `TIMESTAMPTZ`

**重要：时间戳迁移决策**
- 现有模型用 `BigInteger` 毫秒时间戳，架构文档要求 `TIMESTAMP WITH TIME ZONE`
- 本 Story 的新模型（RefreshToken/AuditLog）**必须使用 `TIMESTAMPTZ`**，与架构对齐
- User 模型重构时一并迁移 `created_at`/`updated_at` 为 `TIMESTAMPTZ`
- 使用 `qysp.utils.time` 中的 `now_beijing()`/`to_beijing()` 处理时区

### 认证方案详细规范

**JWT 双令牌机制：**

| Token | 有效期 | 存储位置 | 用途 |
|-------|--------|---------|------|
| Access Token | 15 分钟 | 前端内存（不存 localStorage） | API 请求 `Authorization: Bearer <token>` |
| Refresh Token | 30 天 | HttpOnly Cookie（SameSite=Strict） | 无感刷新 Access Token |

**Refresh Token 安全存储：**
- 数据库 `refresh_tokens` 表存 `token_hash`（SHA-256 哈希），**不存明文**
- 吊销时设置 `revoked_at` 字段
- Redis 黑名单存已吊销 Token 的 JTI（用于快速校验）

**验证码 Redis Key 设计：**
```
sms:code:{phone}     → "123456"     TTL=300s（验证码值）
sms:throttle:{phone} → "1"          TTL=60s（防重发标记）
```

### 数据库 Schema — 重构后的 User 模型

```python
class User(db.Model):
    __tablename__ = 'users'
    id = db.Column(db.String, primary_key=True, default=gen_id)
    phone = db.Column(db.String(20), unique=True, nullable=True)  # 手机号，软删除后清空
    nickname = db.Column(db.String(200), nullable=False)
    avatar_url = db.Column(db.String, nullable=True, default='')
    bio = db.Column(db.String(200), nullable=True, default='')
    role = db.Column(db.String, nullable=False, default='user')  # 'user' | 'admin'
    plan_level = db.Column(db.String, nullable=False, default='free')  # free|lite|pro|expert
    is_banned = db.Column(db.Boolean, default=False)
    onboarding_completed = db.Column(db.Boolean, default=False)
    deleted_at = db.Column(db.DateTime(timezone=True), nullable=True)  # 软删除标记
    created_at = db.Column(db.DateTime(timezone=True), server_default=db.func.now())
    updated_at = db.Column(db.DateTime(timezone=True), server_default=db.func.now(), onupdate=db.func.now())
```

```python
class RefreshToken(db.Model):
    __tablename__ = 'refresh_tokens'
    id = db.Column(db.String, primary_key=True, default=gen_id)
    user_id = db.Column(db.String, db.ForeignKey('users.id'), nullable=False)
    token_hash = db.Column(db.String(64), nullable=False)  # SHA-256 哈希
    expires_at = db.Column(db.DateTime(timezone=True), nullable=False)
    revoked_at = db.Column(db.DateTime(timezone=True), nullable=True)
    created_at = db.Column(db.DateTime(timezone=True), server_default=db.func.now())
```

### 安全要求 Checklist

- [ ] 验证码：6 位随机数字，Redis TTL 5 分钟，60s 防重发
- [ ] Refresh Token：HttpOnly + SameSite=Strict Cookie，数据库存哈希
- [ ] 手机号脱敏：API 响应中返回 `138****8000` 格式
- [ ] 输入验证：手机号正则 `^1[3-9]\d{9}$`，验证码 `^\d{6}$`
- [ ] 防暴力破解：验证码错误超过 5 次锁定手机号 30 分钟
- [ ] 审计日志：注册操作写入 `audit_logs` 表

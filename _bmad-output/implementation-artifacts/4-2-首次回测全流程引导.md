# Story 4.2: 首次回测全流程引导（≤5 步）

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

作为新注册用户，
我希望引导流程直接带我完成第一次回测，
以便在 5 步以内体验到 QYQuant 的核心价值。

## Acceptance Criteria

1. **Given** 新手引导第 2 步"一键回测策略"被触发
   **When** 用户按引导步骤操作
   **Then** 操作路径为：进入策略广场选择精选策略（1步）→ 确认回测参数（1步）→ 点击运行（1步）→ 查看报告（1步），总计 ≤4 步（NFR18，≤5 步）（FR8）
   **And** 每步有明确视觉引导（高亮或箭头），告知用户下一步做什么
   **And** 引导使用的预置策略回测成功率 ≥95%（NFR24）
   **And** 回测执行时显示进度动画并提示预计等待时间（NFR25）

## Dev Notes

### 架构约束与模式

- **前端框架**: Vue 3 + Composition API + `<script setup>` + Pinia 状态管理
- **后端框架**: Flask + 蓝图组织，所有路由前缀 `/api/v1/`
- **数据库**: PostgreSQL，使用 Alembic 管理 migration
- **认证**: JWT Access Token（内存）+ Refresh Token（HttpOnly Cookie）
- **时区规范**: 数据库存 UTC（TIMESTAMP WITH TIME ZONE），API 输出 ISO 8601 带 +08:00，禁止 `datetime.now()`
- **回测 API**: `POST /api/v1/backtest/` — 复用现有回测提交流程
- **沙箱执行**: E2B Cloud SDK 封装，策略代码在 Firecracker microVM 中执行

### UX 设计要求

- **核心目标**: 从"引导触发"到"看到报告" ≤4 步操作，≤25 秒等待
- **渐进式信息披露**（UX-3）：引导中每步只展示当前需要的信息
- **等待即体验**（情感原则 #2）：回测运行期间分 3 阶段进度提示，转化等待为仪式感
- **成功值得庆祝**（情感原则 #4）：首次回测成功后轻微庆祝动效
- **零代码路径**: 小白用户全程不需要看到任何代码
- **设计令牌**:
  - 卡片圆角: 12px，内边距: 20px，间距: 16px
  - 标题字号: 20px / 600wt，正文字号: 16px / 400wt
  - CTA 按钮色: #1A1A1A（黑色），8px 圆角
  - 导航激活/步骤指示: #F5E642（黄色胶囊）
  - 涨色: #E53935 红 / 跌色: #26A69A 绿
  - 核心指标: 32px / 700wt 超大数字

### 前端组件规范

- **组件路径**: `frontend/src/components/onboarding/GuidedBacktestFlow.vue`
- **高亮实现**: 复用 Story 4.1 的 overlay + `clip-path` 镂空方案（`position: fixed; z-index: 1000`）
- **状态管理**: Pinia `useUserStore()` 追踪引导步骤，`useBacktestStore()` 管理回测状态
- **进度组件**: 若 `BacktestProgress` 组件已在 Epic 3 中创建则直接复用，否则内联实现 3 阶段进度（数据加载/策略运行/生成报告）
- **复用组件**: 策略卡片 `StrategyCard`（如存在）、`BacktestCard`、`StatCard`、`EquityCurveChart`（如存在）
- **性能目标**: 引导步骤切换 <200ms，首屏加载 <500ms

### 后端 API 规范

- **响应格式统一**:
  ```json
  { "data": { ... }, "meta": {} }
  ```
- **错误格式**:
  ```json
  { "error": { "code": "BACKTEST_FAILED", "message": "...", "details": {} } }
  ```
- **回测提交**: 复用 `POST /api/v1/backtest/` — 传入 strategy_id + params，返回 job_id
- **回测轮询**: 复用 `GET /api/v1/backtest/:job_id` — 返回状态和结果

### 无障碍要求（WCAG 2.1 AA）

- 文本对比度 ≥4.5:1（`#1E293B` on `#FFFFFF`）
- 交互元素最小尺寸 44×44px
- 完整 Tab 键导航支持（引导步骤间 Tab 切换）
- Focus 样式: `2px solid #E53935` + `outline-offset: 2px`
- 所有交互元素添加 `aria-label`
- 进度反馈使用 `aria-live="polite"` 动态更新
- 支持 `prefers-reduced-motion`：禁用动画，保留功能

### 关键依赖

- **前置依赖（必须完成）**:
  - Story 4.1（OnboardingGuide 组件框架 — `ready-for-dev`）
  - Epic 2 用户认证系统（`users` 表、JWT 认证、`GET /api/v1/auth/profile`）
  - Epic 3 回测 API（`POST /api/v1/backtest/`、回测结果查询）
- **可选依赖（有则复用，无则简化实现）**:
  - `BacktestProgress` 组件（Epic 3-5）
  - `EquityCurveChart` 组件（Epic 3-5）
  - `StrategyCard` 组件（Epic 5-1）
  - 策略广场页面和 API（Epic 5）
- **依赖降级策略**: 若 Epic 3/5 相关组件尚未实现，引导流程可使用简化版：直接展示预置策略信息卡片 → 一键回测 → 展示结果，不依赖策略广场页面路由

### 防踩坑提醒

- **不要** 创建独立的"引导回测" API — 必须复用现有 `POST /api/v1/backtest/` 接口，确保引导体验与正常流程完全一致
- **不要** 硬编码预置策略 ID — 使用 tag/标记查询，策略 ID 可能因环境不同而变化
- **不要** 在引导流程中跳过额度检查 — 引导回测也消耗回测额度，保持一致性
- **不要** 使用 localStorage 存储引导进度 — 存在 Pinia store 中（内存），跨会话状态由 `users.onboarding_completed` 数据库字段控制
- **不要** 自行实现回测轮询逻辑 — 复用 `useBacktestStore` 已有的轮询机制
- **不要** 在引导完成后自动导航离开 — 让用户自行选择"导入策略库"或"返回首页"

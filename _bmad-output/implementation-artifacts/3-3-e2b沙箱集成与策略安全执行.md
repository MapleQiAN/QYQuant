# Story 3.3: E2B 沙箱集成与策略安全执行

Status: done

## Story

作为平台，
我希望通过 E2B Cloud 在 Firecracker microVM 沙箱中执行策略代码，
以便策略代码在完全隔离的环境中运行，禁止网络访问和文件系统写入。

## 验收标准

1. **策略代码解密与沙箱执行**
   - Given strategies 表中有 AES-256-GCM 加密存储的策略代码
   - When Worker 执行回测任务
   - Then Flask 解密策略代码 → 传入 E2B 沙箱执行 → 执行完毕后沙箱销毁，明文不落盘

2. **网络隔离**
   - And 沙箱内禁止网络访问（NFR10/FR63）

3. **文件系统隔离**
   - And 沙箱内禁止文件系统写入（NFR10/FR63）

4. **超时控制**
   - And 单次回测最大执行时间 5 分钟，超时后任务状态更新为 timeout（FR64）

5. **性能要求**
   - And 简单策略（单标的/1年/日线）在已缓存时 10 秒内完成（NFR2）

6. **预热容器池**
   - And 系统维持预热容器池，冷启动时间 ≤1 秒（NFR28）

### 关键技术细节

**AES-256-GCM 加密示例：**
```python
from cryptography.hazmat.primitives.ciphers.aead import AESGCM
import os

def encrypt_strategy(plaintext: bytes, key: bytes) -> bytes:
    aesgcm = AESGCM(key)
    nonce = os.urandom(12)
    ct = aesgcm.encrypt(nonce, plaintext, None)
    return nonce + ct  # 12 bytes nonce + ciphertext

def decrypt_strategy(data: bytes, key: bytes) -> bytes:
    aesgcm = AESGCM(key)
    nonce, ct = data[:12], data[12:]
    return aesgcm.decrypt(nonce, ct, None)
```

**E2B 集成示例：**
```python
from e2b_code_interpreter import Sandbox

def execute_in_sandbox(code: str, data: dict, params: dict) -> dict:
    sandbox = Sandbox(api_key=E2B_API_KEY, timeout=300)
    try:
        # 注入数据和参数
        sandbox.run_code(f"import json; data = json.loads('{json.dumps(data)}')")
        sandbox.run_code(f"params = json.loads('{json.dumps(params)}')")
        # 执行用户策略
        result = sandbox.run_code(code)
        return parse_result(result)
    finally:
        sandbox.close()
```

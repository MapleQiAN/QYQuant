# Story 5.3: 一键导入策略库并发起回测试用

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

作为用户，
我希望将策略广场中的策略一键导入我的策略库，并立即发起回测试用，
以便在 2 步内体验"策略即服务"的核心流程。

## Acceptance Criteria

1. **Given** 用户在策略详情页点击"免费回测试用"
   **When** 调用 `POST /api/v1/marketplace/strategies/:id/import`
   **Then** 在用户的 strategies 中创建关联记录（`source='marketplace'`，FR29）
   **And** 自动跳转至回测参数配置页（复用 Epic 3 参数配置器，FR30）
   **And** 操作总步骤 ≤2 步（点击导入 → 确认参数）
   **And** 已导入过的策略显示"已在策略库中"，不重复创建记录

## Dev Notes

### 架构约束与模式

- **前端框架**: Vue 3 + Composition API + TypeScript + Pinia
- **后端框架**: Flask + 蓝图，路由前缀 `/api/v1/`
- **数据库**: PostgreSQL + Alembic
- **认证**: `@require_auth` 装饰器，JWT 验证
- **API 响应格式**: `{ "data": {...}, "meta": {...} }`

### 导入机制设计

- **不复制代码**: 导入的策略不复制 `code_encrypted`，而是通过 `source_strategy_id` 引用原始策略
- **回测时**: 后端通过 `source_strategy_id` 找到原始策略的代码执行回测
- **用户体验**: 导入后用户在"我的策略库"中看到该策略，可直接发起回测
- **操作路径**: 点击"免费回测试用" → 策略自动导入 → 跳转回测配置页（≤2 步）

### 数据库变更

```sql
ALTER TABLE strategies ADD COLUMN source VARCHAR(20) DEFAULT 'upload';
ALTER TABLE strategies ADD COLUMN source_strategy_id INTEGER REFERENCES strategies(id);
ALTER TABLE strategies ADD CONSTRAINT uq_user_imported_strategy UNIQUE(author_id, source_strategy_id);
```

### 防踩坑提醒

- **不要** 复制策略代码到新记录 — 使用 `source_strategy_id` 引用
- **不要** 允许未认证用户调用导入 API — 必须 `@require_auth`
- **不要** 忽略重复导入检查 — 使用数据库唯一约束 + 应用层检查
- **不要** 在导入时触发审核流程 — 导入的策略是个人使用，无需审核
- **跳转路径**: 确保回测配置页路由存在（依赖 Epic 3），若不存在则跳转到策略库页面

# Story 7.1: 创建模拟托管机器人与免责提示

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

作为已登录用户，
我希望从策略库中选择策略创建模拟托管机器人，并在首次进入时看到免责提示，
以便开始模拟交易前了解风险。

## Acceptance Criteria

1. **Given** 用户首次进入模拟托管页面（`users.sim_disclaimer_accepted = false`）
   **When** 页面加载
   **Then** 弹出一次性免责提示弹窗，用户勾选"我已知晓"后关闭，写入 `sim_disclaimer_accepted = true`（FR46）
   **And** 此后进入该页面不再弹出（前端通过 `useAuthStore` 中的 `currentUser.sim_disclaimer_accepted` 判断）

2. **Given** 用户点击"创建机器人"并选择策略、输入初始资金
   **When** 提交 `POST /api/v1/simulation/bots`
   **Then** `simulation_bots` 表创建记录（`status='active'`，FR40/FR41）
   **And** 返回新建机器人信息（id、strategy_id、initial_capital、status、created_at）

3. **Given** 用户当前 `status='active'` 的机器人数已达套餐上限（`free=1, lite=2, pro=3, expert=5`）
   **When** 提交 `POST /api/v1/simulation/bots`
   **Then** 返回 `403` 和错误码 `SIMULATION_SLOT_LIMIT_REACHED`，提示升级套餐（FR48）

4. **Given** 用户选择的策略不在自己的策略库中
   **When** 提交 `POST /api/v1/simulation/bots`
   **Then** 返回 `404` 和错误码 `STRATEGY_NOT_FOUND`

5. **Given** 用户未登录
   **When** 访问任何 `/api/v1/simulation/*` 端点
   **Then** 返回 `401 Unauthorized`

  - [ ] 4.3 实现 `POST /api/v1/simulation/bots`（创建机器人）：
    ```python
    @bp.post('/bots')
    @jwt_required()
    def create_bot():
        user_id = get_jwt_identity()
        user = User.query.get(user_id)
        if not user or user.deleted_at:
            return error_response('USER_NOT_FOUND', '用户不存在', 404)

        payload = request.get_json() or {}
        strategy_id = payload.get('strategy_id')
        initial_capital = payload.get('initial_capital')

        # 验证策略属于当前用户
        strategy = Strategy.query.filter_by(
            id=strategy_id, owner_id=user_id
        ).first()
        if not strategy:
            return error_response('STRATEGY_NOT_FOUND', '策略不存在或无权限', 404)

        # 检查槽位限制
        slot_limit = SLOT_LIMITS.get(user.plan_level, 1)
        active_count = SimulationBot.query.filter_by(
            user_id=user_id, status='active'
        ).count()
        if active_count >= slot_limit:
            return error_response(
                'SIMULATION_SLOT_LIMIT_REACHED',
                f'当前套餐最多支持 {slot_limit} 个运行中的机器人，请升级套餐',
                403
            )

        bot = SimulationBot(
            user_id=user_id,
            strategy_id=strategy_id,
            initial_capital=initial_capital,
            status='active',
        )
        db.session.add(bot)
        db.session.commit()

        return ok({
            'id': bot.id,
            'strategy_id': bot.strategy_id,
            'initial_capital': str(bot.initial_capital),
            'status': bot.status,
            'created_at': format_beijing_iso(bot.created_at),
        }), 201
    ```

## Dev Notes

### 架构约束与模式

- **后端框架**: Flask + flask-smorest，`@bp.post()` / `@bp.get()` 装饰器
- **认证**: 所有 `/api/v1/simulation/` 端点均需 `@jwt_required()`（`get_jwt_identity()` 获取 user_id）
- **响应格式**: `ok(data)` 封装成功响应，`error_response(code, message, status)` 封装错误
- **时区**: `created_at` 用 `format_beijing_iso(datetime_obj)` 转换为北京时间 ISO 字符串
- **前端状态管理**: Pinia store，组件级加载用 `ref(false)`，全局加载用 store state
- **TypeScript**: 严格类型，`SimulationBot.initial_capital` 为 `string`（后端 Numeric → JSON string）

### ⚠️ 关键陷阱：旧版 `bots.py` / `BotInstance` 是 LEGACY，不要修改

**现有状态：**
- `backend/app/blueprints/bots.py` — 注册在 `/api/bots`（无 `/v1`），使用 `BotInstance` 模型
- `backend/app/models.py` 中的 `BotInstance` — 对应 `bot_instances` 表（字段不同：`name`, `strategy`字符串, `profit`, `runtime` 等）
- `frontend/src/stores/bots.ts` — 旧 store，对应旧 `/api/bots` 端点

**Story 7.1 做的是全新实现：**
- 新蓝图 `simulation.py` → `/api/v1/simulation/`
- 新模型 `SimulationBot` → `simulation_bots` 表（含 `strategy_id` FK、`initial_capital` Numeric、`status` 标准化）
- 新 store `useSimulationStore.ts`（独立于旧 `bots.ts`）

**绝对不要**：修改或删除 `bots.py`、`BotInstance` 模型、`bots.ts` store

### ⚠️ 关键陷阱：Strategy.owner_id vs Strategy.user_id

检查 `backend/app/models.py` 中 `Strategy` 模型的用户 FK 字段名是 `owner_id` 还是 `user_id`：
```python
# 根据实际字段名选择：
strategy = Strategy.query.filter_by(id=strategy_id, owner_id=user_id).first()
# 或：
strategy = Strategy.query.filter_by(id=strategy_id, user_id=user_id).first()
```
**不要假设字段名**，必须先读 `models.py` 确认。

### ⚠️ 关键陷阱：authStore 中 currentUser 的来源

前端 `BotsView.vue` 依赖 `authStore.currentUser.sim_disclaimer_accepted` 判断是否显示免责弹窗。**必须确认**：
1. `GET /api/v1/users/me` 返回 `sim_disclaimer_accepted` 字段（Task 5）
2. `authStore.currentUser` 由 `GET /api/v1/users/me` 填充（检查现有 `user.ts` store 中 `fetchMe` 函数）
3. 用户接受免责声明后调用 `authStore.fetchMe()` 刷新用户信息

### ⚠️ 关键陷阱：`initial_capital` 类型

- 后端 `SimulationBot.initial_capital` 是 `db.Numeric(18, 2)`
- PostgreSQL NUMERIC 序列化为 JSON 时是 **字符串**（`"100000.00"`）
- 前端 `SimulationBot.initial_capital` 类型定义为 `string`，**不要用 `number`**
- 前端创建表单中用户输入的 `initial_capital` 是 `number`，POST 请求体中以 number 发送
- 后端接收 `initial_capital` 时用 `Decimal(str(payload['initial_capital']))` 或直接赋值给 SQLAlchemy Numeric 列

```python
slot_limit = SLOT_LIMITS.get(user.plan_level, 1)  # 默认 free=1
active_count = SimulationBot.query.filter_by(
    user_id=user_id, status='active'  # 仅 active 计入，paused 不计
).count()
if active_count >= slot_limit:
    return error_response('SIMULATION_SLOT_LIMIT_REACHED', ..., 403)
```

**注意**：`status='paused'` 的机器人**不占用**槽位（用户可将 active 暂停后创建新的）

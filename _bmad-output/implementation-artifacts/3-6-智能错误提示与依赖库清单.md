# Story 3.6: 智能错误提示与依赖库清单

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

作为已登录用户，
我希望策略代码执行出错时看到智能错误提示，并能查看平台支持的依赖库清单，
以便快速定位问题并修复代码。

## 验收标准

1. **智能错误提示**
   - Given 回测任务 status=failed（用户代码错误）
   - When 用户查看失败报告
   - Then 错误提示包含具体行号、错误类型和可读说明（NFR21），不是通用错误码

2. **常见错误示例代码**
   - And 对常见错误（缺少参数、未定义变量、类型错误）提供示例代码链接（FR21）

3. **支持的依赖库清单**
   - Given 用户点击"查看支持的依赖库"
   - When 访问 `GET /api/v1/backtest/supported-packages`
   - Then 返回平台白名单库列表（pandas、numpy、ta-lib 等）及版本号（FR24）

## Dev Notes

### 架构模式与约束

- **错误解析**：从 E2B 沙箱的 stderr 提取结构化信息，不暴露沙箱内部路径
- **国际化**：错误说明使用中文（communication_language: 简体中文）
- **安全**：过滤错误信息中的敏感路径、环境变量等
- **白名单库**：与 E2B 沙箱内预装库保持一致

### 现有代码分析（关键！）

1. **`backend/app/tasks/backtests.py`**：Story 3.1/3.3 已有错误处理框架
   - 本 Story 增强错误信息的结构化和可读性

2. **`backend/app/models.py`** 中 `BacktestJob`：
   - `error_message TEXT` 字段已在 Story 3.1 创建
   - 本 Story 改为存储 JSON 格式的结构化错误

3. **前端 `BacktestResult.vue`**：Story 3.5 创建
   - 本 Story 在 failed 状态下展示错误信息

### 关键技术细节

**错误 JSON 结构：**
```json
{
  "type": "NameError",
  "line": 15,
  "message": "未定义的变量 'sma_period'",
  "suggestion": "请检查变量名是否正确，或确认是否在策略参数中定义了该参数",
  "example_code": "# 在 on_init() 中获取参数\nsma_period = ctx.params.get('sma_period', 20)"
}
```

# Story 5.4: 策略发布（免费分享模式）

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

作为已登录用户，
我希望将我的策略发布到策略广场并设置展示指标，发布前了解代码保护机制，
以便与社区分享策略而无需担心代码泄露。

## Acceptance Criteria

1. **Given** 用户选择策略库中的策略并点击"发布到策略广场"
   **When** 填写名称、描述、标签、分类，设置 `display_metrics`
   **Then** 发布前弹出代码保护说明和知识产权协议（FR33）
   **And** 用户确认后 `strategies.review_status` 设为 `pending`，调用 `POST /api/v1/marketplace/strategies`
   **And** 系统发送站内通知"策略已提交审核"（FR59）
   **And** `review_status=approved` 后 `is_public=true`，策略在广场可见（FR31）

## Dev Notes

### 架构约束与模式

- **前端框架**: Vue 3 + Composition API + TypeScript + Pinia
- **后端框架**: Flask + 蓝图，路由前缀 `/api/v1/`
- **数据库**: PostgreSQL + Alembic
- **认证**: `@require_auth` + 所有者验证（策略的 `author_id` 必须等于当前用户）

### 策略发布流程状态机

```
用户策略（私有）
  ↓ 点击"发布到策略广场"
填写信息表单
  ↓
代码保护说明 + IP 协议确认
  ↓ 用户确认
review_status = 'pending'
  ↓ 管理员审核（Epic 10）
  ├─ 通过 → review_status = 'approved', is_public = true
  └─ 拒绝 → review_status = 'rejected', 通知用户
```

### 安全要求

- **代码保护**: 发布后策略代码使用 AES-256-GCM 加密存储（NFR11），仅沙箱执行时解密
- **IP 协议**: 发布前必须展示代码保护说明和知识产权协议（FR33）
- **审核流程**: 发布不自动公开，必须经过管理员审核（FR31）
- **审计日志**: 发布操作记录到 `audit_logs` 表（NFR29）

### display_metrics 格式

```json
{
  "sharpe_ratio": 1.45,
  "max_drawdown": -15.2,
  "total_return": 45.6,
  "win_rate": 62.3,
  "annual_return": 24.8,
  "profit_loss_ratio": 2.1
}
```

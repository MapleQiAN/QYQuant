# Story 5.1: 策略广场列表与精选推荐位

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

作为用户，
我希望浏览策略广场中的策略列表，并看到人工运营的精选推荐位，
以便快速发现值得试用的优质策略。

## Acceptance Criteria

1. **Given** 用户进入策略广场页面
   **When** 访问 `GET /api/v1/marketplace/strategies`
   **Then** 页面首屏 ≤2 秒加载（NFR4），展示卡片式策略列表（FR25）
   **And** 页面顶部有精选推荐位（最多 6 个），展示运营标记为 `is_featured=true` 的策略（FR28）
   **And** 策略卡片展示：名称、分类标签、核心指标摘要、平台验证徽章（`is_verified=true` 时，FR27）
   **And** 支持分页加载，每页 20 条

## Dev Notes

### 架构约束与模式

- **前端框架**: Vue 3 + Composition API + TypeScript + Vite + Pinia 状态管理
- **后端框架**: Flask + 蓝图组织，所有路由前缀 `/api/v1/`
- **数据库**: PostgreSQL，使用 Alembic 管理 migration
- **认证**: JWT Access Token（内存）+ Refresh Token（HttpOnly Cookie）
- **时区规范**: 数据库存 UTC（TIMESTAMPTZ），API 输出 ISO 8601 带 +08:00，禁止 `datetime.now()`
- **API 响应格式**: `{ "data": {...}, "meta": {...} }`
- **错误格式**: `{ "error": { "code": "...", "message": "...", "details": {} } }`

### UX 设计要求

- **极简金融仪表盘风格**（NFR26）：白色主背景、卡片式布局
- **导航模式**: 顶部胶囊 Tab（回测/托管/广场），激活态黄色 `#F5E642`
- **策略卡片规范** (StrategyCard):
  - 高度约 260px
  - 卡片圆角: 12px，内边距: 18px
  - 缩略曲线: 60px 高，红色 `#E53935` 线条
  - 指标值: 14px/700wt，红色表示正收益，深灰表示负值
  - Hover: `transform: translateY(-2px)` + 阴影 `0 4px 12px rgba(0,0,0,0.08)`
- **筛选 Chip**: 胶囊形，激活态黄色 `#F5E642`（本 Story 暂不实现筛选，Story 5.5 实现）
- **色彩系统**:
  - CTA 按钮: `#1A1A1A`（黑色）
  - 验证徽章: `#1890FF`（蓝色）
  - 涨色: `#E53935`（红色），跌色: `#26A69A`（绿色）
  - 主文本: `#1E293B`，次要文本: `#64748B`，辅助文本: `#94A3B8`
- **响应式断点**:
  - `< 768px`: 1 列堆叠
  - `768-1023px`: 2 列网格
  - `≥ 1024px`: 3 列网格

### 数据库 Schema

```sql
-- strategies 表扩展字段（本 Story 新增）
ALTER TABLE strategies ADD COLUMN title VARCHAR(200);
ALTER TABLE strategies ADD COLUMN description TEXT;
ALTER TABLE strategies ADD COLUMN tags TEXT[];
ALTER TABLE strategies ADD COLUMN category VARCHAR(50); -- 'trend-following','mean-reversion','momentum','multi-indicator','other'
ALTER TABLE strategies ADD COLUMN is_public BOOLEAN DEFAULT false;
ALTER TABLE strategies ADD COLUMN is_featured BOOLEAN DEFAULT false;
ALTER TABLE strategies ADD COLUMN is_verified BOOLEAN DEFAULT false;
ALTER TABLE strategies ADD COLUMN review_status VARCHAR(20) DEFAULT 'pending'; -- 'pending','approved','rejected'
ALTER TABLE strategies ADD COLUMN display_metrics JSONB; -- {"sharpe_ratio":1.45,"max_drawdown":-15.2,"total_return":45.6,"win_rate":62.3}

-- 索引
CREATE INDEX idx_strategies_public_verified ON strategies(is_public, is_verified) WHERE is_public = true;
CREATE INDEX idx_strategies_category ON strategies(category);
CREATE INDEX idx_strategies_tags ON strategies USING GIN(tags);
CREATE INDEX idx_strategies_featured ON strategies(is_featured) WHERE is_featured = true;
```

### 防踩坑提醒

- **不要** 在列表 API 中返回策略代码（`code_encrypted`）— 仅返回展示用字段
- **不要** 自行实现分页逻辑 — 使用 SQLAlchemy `paginate()` 或 `offset/limit`
- **不要** 在前端硬编码策略分类 — 从后端获取或使用枚举常量
- **不要** 创建新的 HTTP 客户端 — 复用项目已有的 axios 实例
- **不要** 使用 `datetime.now()` — 使用项目工具库 `now_beijing()` 或 UTC 存储
- **不要** 使用 localStorage 缓存策略列表 — Pinia store 即可
- **display_metrics 字段**: 是 JSONB 类型，直接从 strategies 表读取，不需要实时计算

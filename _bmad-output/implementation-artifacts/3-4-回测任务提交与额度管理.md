# Story 3.4: 回测任务提交与额度管理

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

作为已登录用户，
我希望提交回测任务时系统自动校验和扣减额度，并在界面实时看到排队状态，
以便了解何时可以获得结果，以及剩余可用次数。

## 验收标准

1. **回测任务提交 API**
   - Given 用户点击"开始回测"
   - When 调用 `POST /api/v1/backtest/`（含策略 ID、标的列表、时间区间参数）
   - Then 服务端检查 user_quota.used_count < 套餐限额，否则返回 429 并提示"额度已用完，请升级套餐"

2. **任务创建与入队**
   - And 验证通过后创建 backtest_jobs 记录（status=pending），返回 job_id

3. **额度扣减时机**
   - And 任务开始执行时（Worker 侧）扣减 used_count + 1（避免队列积压虚耗）

4. **任务状态轮询**
   - Given 回测任务排队中
   - When 前端轮询 `GET /api/v1/backtest/:job_id`
   - Then 返回 status 和预估等待时间（基于队列长度估算，FR57）

5. **额度信息展示**
   - And 用户可在回测界面随时查看剩余额度和计费规则说明（FR23）

## 任务 / 子任务

- [ ] Task 1: 创建回测提交 API (AC: #1, #2)
  - [ ] 1.1 在 `backend/app/blueprints/backtest.py` 中创建（或扩展）`POST /api/v1/backtest/` 端点
  - [ ] 1.2 请求体验证：strategy_id（必需）、symbols（列表）、start_date、end_date、parameters（可选 JSONB）
  - [ ] 1.3 验证策略存在且属于当前用户（或为公共策略）
  - [ ] 1.4 额度检查：查询 user_quota 表，used_count < 套餐限额
  - [ ] 1.5 额度不足返回 HTTP 429 + `{"error": {"code": "QUOTA_EXCEEDED", "message": "额度已用完，请升级套餐"}}`
  - [ ] 1.6 创建 BacktestJob 记录（status='pending'），返回 `{"job_id": "..."}`
  - [ ] 1.7 发送 Celery 任务到 backtest 队列

- [ ] Task 2: 实现 Worker 侧额度扣减 (AC: #3)
  - [ ] 2.1 在 `backend/app/tasks/backtests.py` 任务开始执行时（status → running），扣减 used_count + 1
  - [ ] 2.2 使用数据库事务确保扣减原子性
  - [ ] 2.3 如果扣减时发现额度已满（竞态条件），任务回退为 failed

- [ ] Task 3: 创建任务状态查询 API (AC: #4)
  - [ ] 3.1 创建 `GET /api/v1/backtest/:job_id` 端点

- [ ] Task 4: 创建额度查询 API (AC: #5)
  - [ ] 4.1 创建 `GET /api/v1/backtest/quota` 端点
  - [ ] 4.2 返回字段：used_count、plan_limit、plan_level、reset_at
  - [ ] 4.3 plan_limit 根据 plan_level 计算（free=10, basic=100, pro=500, enterprise=unlimited）

- [ ] Task 5: 实现套餐限额配置 (AC: #1, #5)
  - [ ] 5.1 创建 `backend/app/config/quota.py`，定义各套餐限额
  - [ ] 5.2 限额常量：PLAN_LIMITS = {"free": 10, "basic": 100, "pro": 500, "enterprise": float('inf')}
  - [ ] 5.3 新用户注册时自动创建 user_quota 记录（plan_level='free', used_count=0）

- [ ] Task 6: 编写测试 (AC: #1-#5)
  - [ ] 6.1 测试回测提交成功：创建 job + 返回 job_id
  - [ ] 6.2 测试额度不足：返回 429
  - [ ] 6.3 测试 Worker 侧额度扣减
  - [ ] 6.4 测试任务状态查询
  - [ ] 6.5 测试额度查询 API
  - [ ] 6.6 测试未认证用户无法提交回测（401）

## Dev Notes

### 架构模式与约束

- **API 风格**：RESTful，`/api/v1/backtest/` 前缀
- **认证**：JWT Bearer Token（Epic 2 实现），本 Story 依赖 `@jwt_required()` 装饰器
- **错误格式**：统一 `{"error": {"code": "...", "message": "...", "details": {...}}}`
- **额度扣减时机**：任务**开始执行**时扣减（Worker 侧），避免队列积压虚耗
- **蓝图**：`backend/app/blueprints/backtest.py`（已有文件，需扩展或重构）

**预估等待时间算法：**
```python
pending_count = BacktestJob.query.filter_by(status='pending').count()

estimated_wait = pending_count * avg_execution_time
```

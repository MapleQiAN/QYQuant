# Story 8.2: 选择套餐并发起支付

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

作为已登录用户，
我希望选择套餐并通过微信支付或支付宝完成付费，
以便解锁更多回测次数和模拟托管槽位。

## Acceptance Criteria

1. **Given** 用户访问套餐升级页面（`/pricing`）
   **When** 页面加载完成
   **Then** 展示 4 级套餐对比表：免费（free，10次/月）/ 轻量（lite，200元/月，200次/月）/ 进阶（pro，500元/月，500次/月）/ 专业（expert，1000元/月，无限次/月）（FR4）
   **And** 当前套餐高亮显示，当前套餐不显示支付按钮（仅显示"当前套餐"标签）

2. **Given** 用户选择目标套餐并点击"微信支付"或"支付宝"按钮
   **When** 调用 `POST /api/v1/payments/orders`（body: `{ plan_level, provider }`）
   **Then** 后端创建 `payment_orders` 记录（status=pending），返回 `{ order_id, pay_url }`
   **And** 若用户已有同 plan_level 的 pending 订单，返回已有订单（幂等处理）
   **And** 支付宝返回收银台链接；微信支付返回二维码 URL 或 h5 跳转链接（FR5）
   **And** 未登录时返回 `401 Unauthorized`
   **And** 传入无效 plan_level（如 "basic"、"enterprise"）返回 `400 Bad Request`（明确提示有效值）

3. **Given** 用户发起支付请求成功
   **When** 前端收到 `{ order_id, pay_url }` 响应
   **Then** 显示支付金额、套餐内容和服务条款确认提示（NFR14）
   **And** 微信支付：展示二维码图片或"打开微信支付"跳转链接
   **And** 支付宝：跳转至支付宝收银台链接（新标签页打开）

4. **Given** `BacktestsView.vue` 中的"升级套餐解锁更多次数"按钮
   **When** 用户点击该禁用引导按钮（Story 8.1 实现）
   **Then** 路由跳转至 `/pricing`

    # 套餐价格（人民币元）
    PLAN_PRICES = {
        "free": 0,
        "lite": 200,
        "pro": 500,
        "expert": 1000,
    }

    VALID_PROVIDERS = {"wechat", "alipay"}


    def _get_user_or_404(user_id):
        user = db.session.get(User, user_id)
        if user is None or user.deleted_at is not None:
            return None, error_response('NOT_FOUND', '用户不存在', 404)
        return user, None


    @bp.post('/orders')
    @jwt_required()
    def create_payment_order():
        user, err = _get_user_or_404(get_jwt_identity())
        if err:
            return err

        data = request.get_json(silent=True) or {}
        plan_level = normalize_plan_level(data.get('plan_level', ''))
        provider = data.get('provider', '')

        # 校验 plan_level
        if plan_level not in PLAN_LEVELS or plan_level == 'free':
            return error_response(
                'INVALID_PLAN',
                f"无效套餐等级，有效值为: {', '.join(p for p in PLAN_LEVELS if p != 'free')}",
                400,
            )

        # 校验 provider
        if provider not in VALID_PROVIDERS:
            return error_response(
                'INVALID_PROVIDER',
                f"无效支付方式，有效值为: {', '.join(VALID_PROVIDERS)}",
                400,
            )

        # 幂等检查：如果存在同用户同 plan_level 的 pending 订单，直接返回
        existing = (
            PaymentOrder.query
            .filter_by(user_id=user.id, plan_level=plan_level, status='pending')
            .order_by(PaymentOrder.created_at.desc())
            .first()
        )
        if existing:
            return ok({
                'order_id': existing.id,
                'pay_url': existing.pay_url,
                'plan_level': existing.plan_level,
                'amount': float(existing.amount),
                'provider': existing.provider,
            })

        # 生成支付链接（沙箱/开发环境下使用 mock URL）
        amount = PLAN_PRICES[plan_level]
        pay_url = _generate_pay_url(provider, plan_level, amount)

        order = PaymentOrder(
            user_id=user.id,
            plan_level=plan_level,
            amount=amount,
            provider=provider,
            pay_url=pay_url,
            status='pending',
        )
        db.session.add(order)
        db.session.commit()

        return ok({
            'order_id': order.id,
            'pay_url': order.pay_url,
            'plan_level': order.plan_level,
            'amount': float(order.amount),
            'provider': order.provider,
        }), 201


    def _generate_pay_url(provider: str, plan_level: str, amount: int) -> str:
        """
        生成支付 URL。
        生产环境：调用 wechatpayv3 或 alipay-sdk-python 生成真实支付链接。
        开发/沙箱环境：返回占位 mock URL（由 PAYMENT_SANDBOX env var 控制）。

        Story 8.2 实现沙箱模式；真实 SDK 集成在 Story 8.3 回调处理时补全。
        """
        import os
        if os.environ.get('PAYMENT_SANDBOX', 'true').lower() == 'true':
            # 沙箱模式：返回 mock 支付链接，供前端展示和 Story 8.3 Webhook 测试
            return f"https://pay.mock.example.com/{provider}/{plan_level}?amount={amount}"
        # TODO: Story 8.3 实现真实 SDK 调用
        raise NotImplementedError("生产支付集成待 Story 8.3 实现")

    PLAN_LEVELS = set(PLAN_PRICES.keys())

### ⚠️ 关键陷阱 1：plan_level 命名历史遗留问题（必须处理）

**现状（quota.py）：**
```python
PLAN_LIMITS = {"free": 10, "basic": 100, "pro": 500, "enterprise": math.inf}
```

**架构文档定义：**`plan_level ENUM('free','lite','pro','expert')`

**Epic 8 规划：** 4 级套餐 → `free/lite/pro/expert`（与架构文档一致）

**Story 8.1 处理方式：** 维持现状（已在 8.1 story 文档中记录为技术债）。**Story 8.2 必须统一。**

**正确处理方法：**
1. 在 `quota.py` 中更新 `PLAN_LIMITS`，使用 `lite/expert`
2. 添加 `PLAN_LEVEL_ALIASES` 向后兼容旧名称（`basic→lite`, `enterprise→expert`）
3. **不要**在 Story 8.2 中修改数据库中已有的 `user_quota.plan_level` 字段值——`normalize_plan_level` 会动态处理
4. **不要**在支付订单中创建 `basic` 或 `enterprise` 类型的记录

**影响范围：** 修改 `PLAN_LIMITS` 后，`get_plan_limit("basic")` 将通过别名路由至 `get_plan_limit("lite")` 返回 200，而非原来的 100。**需确认 Story 8.1 测试中的 `plan_limit` 期望值是否需要同步调整（测试用 `free` 套餐不受影响）。**

### ⚠️ 关键陷阱 2：payments.py 蓝图中 PLAN_LEVELS 变量声明顺序

在 `payments.py` 中，`PLAN_LEVELS = set(PLAN_PRICES.keys())` 必须在 `PLAN_PRICES` 字典定义**之后**声明，否则会 NameError。建议将其放在文件末尾，或在函数内部引用 `PLAN_PRICES`。

### ⚠️ 关键陷阱 3：微信支付沙箱 vs 生产环境

Story 8.2 **不集成**真实的 `wechatpayv3` SDK（那是 Story 8.3 的工作）。本 Story 的职责是：
- 创建订单记录
- 返回沙箱 mock pay_url（用于前端展示和 Story 8.3 Webhook 测试）
- 环境变量 `PAYMENT_SANDBOX=true` 控制行为（在 Flask config 或 `.env` 中设置）

**不要**在 Story 8.2 中安装或调用 `wechatpayv3` 或 `alipay-sdk-python` SDK。

### ⚠️ 关键陷阱 4：登录方式与测试 fixture

项目实际使用**手机号验证码登录**（`POST /api/v1/auth/send-code` + `POST /api/v1/auth/login`），而非密码登录。测试中的 `_register_and_login` fixture 需要参考 `backend/tests/test_users.py` 中已有的 `_login_user` helper（它直接插入数据库并通过验证码流程）。先读取该文件确认正确的 test helper 写法，再编写 `test_payments.py`。

### ⚠️ 关键陷阱 5：BacktestsView.vue 的升级按钮修改

Story 8.1 中"升级套餐解锁更多次数"按钮是 `disabled` 的静态按钮。Story 8.2 需要让它可点击并路由到 `/pricing`。**先 Read `frontend/src/views/BacktestsView.vue` 找到精确的按钮代码位置**，不要盲目修改。可以：
- 将 `<button disabled>` 替换为 `<router-link to="/pricing">` 包裹的链接按钮
- 或移除 `disabled` 并添加 `@click="$router.push('/pricing')"`

### ⚠️ 关键陷阱 6：client 导入路径

在 `frontend/src/api/payments.ts` 中，`client` 的正确导入路径需要通过阅读 `frontend/src/api/users.ts` 确认（可能是 `./client`、`../lib/client` 或直接是 axios 实例）。**不要猜测**，先 Read users.ts。

# Story 9.1: 站内通知系统

Status: ready-for-dev

## Story

作为已登录用户，
我希望在平台内收到及时的站内通知，
以便不遗漏任何重要平台消息（策略审核结果、额度提醒等）。

## Acceptance Criteria

1. **Given** `notifications` 表（id、user_id、type、title、content、is_read、created_at）已通过 migration 创建
   **When** 平台触发通知事件（策略审核通过/拒绝、额度用完提醒等）并写入 `notifications` 表（`is_read=false`）
   **Then** 用户导航栏的通知图标显示未读数徽标（FR59）

2. **Given** 前端已初始化通知轮询
   **When** 定时触发（每 30 秒）
   **Then** 调用 `GET /api/v1/notifications/unread-count`，返回 `{"data": {"count": N}}`，自动更新导航栏徽标数字（ARCH-13）

3. **Given** 用户点击通知铃铛图标
   **When** 调用 `GET /api/v1/notifications?page=1&per_page=20`
   **Then** 返回通知列表，按 `created_at` 倒序，每页 20 条，每条含：`id`、`type`、`title`、`content`、`is_read`、`created_at`（ISO 8601 北京时间格式）

4. **Given** 用户点击某条未读通知
   **When** 调用 `PATCH /api/v1/notifications/<id>/read`
   **Then** 该通知 `is_read` 更新为 `true`，返回 `{"data": {"ok": true}}`，前端未读数减 1

5. **Given** 其他业务模块（策略审核、额度扣减等）需要发通知
   **When** 调用内部工具函数 `create_notification(db, user_id, type, title, content)`
   **Then** 在 `notifications` 表插入一条 `is_read=false` 的通知记录

## Dev Notes

### 关键架构约束

- **通知推送方式**：站内通知使用 30 秒轮询，**不使用** WebSocket 或 SSE（ARCH-13）。
- **邮件通知（Story 9.2）不在本 story 范围内**，本 story 仅做站内通知。
- `notifications` 表属于 Epic 9 新增表，**没有**任何历史数据，migration 必须创建全新表。
- `Notification` 模型需添加到 `backend/app/models.py` 现有文件中（该项目所有模型集中在单一 models.py），**不要**创建单独的 notification.py 模型文件。
- 内部工具函数 `create_notification` 需位于 `backend/app/utils/notification.py`，供其他蓝图（如策略审核、支付回调）导入使用，**不要**直接在业务代码中手写 `Notification(...)` 对象。

### 现有代码模式（必须遵循）

**后端 Blueprint 标准写法**（参考 `backend/app/blueprints/users.py`、`simulation.py`）：
```python
from flask_smorest import Blueprint
from ..extensions import db
from ..models import Notification, User
from ..utils.response import error_response, ok
from ..utils.time import format_beijing_iso, now_utc
from flask_jwt_extended import jwt_required, get_jwt_identity

bp = Blueprint('notifications', __name__, url_prefix='/api/v1/notifications')

@bp.get('/unread-count')
@jwt_required()
def unread_count():
    user_id = get_jwt_identity()
    count = Notification.query.filter_by(user_id=user_id, is_read=False).count()
    return ok({'count': count})
```

**后端 Model 标准写法**（参考 `backend/app/models.py` 现有模型）：
```python
class Notification(db.Model):
    __tablename__ = 'notifications'
    __table_args__ = (
        db.Index('ix_notifications_user_id', 'user_id'),
        db.Index('ix_notifications_created_at', 'created_at'),
    )
    id = db.Column(db.String, primary_key=True, default=gen_id)
    user_id = db.Column(db.String, db.ForeignKey('users.id'), nullable=False)
    type = db.Column(db.String(50), nullable=False)
    title = db.Column(db.String(200), nullable=False)
    content = db.Column(db.Text, nullable=True)
    is_read = db.Column(db.Boolean, nullable=False, default=False)
    created_at = db.Column(db.DateTime(timezone=True), nullable=False, default=now_utc)
```

**时间规范**（ARCH-16，强制执行）：
- 数据库列使用 `db.DateTime(timezone=True)`（即 PostgreSQL `TIMESTAMPTZ`）
- 默认值 `default=now_utc`（来自 `backend/app/utils/time.py`）
- API 响应时间使用 `format_beijing_iso(dt)` 输出 `"+08:00"` 格式
- **禁止** `datetime.now()`（无时区）、`datetime.utcnow()`

**响应格式规范**（强制执行）：
- 成功：`return ok(data_dict)` → `{"data": {...}}`
- 错误：`return error_response("ERROR_CODE", "message", status_code)`
- **禁止**直接返回裸 dict

**前端 Store 标准写法**（参考 `frontend/src/stores/useSimulationStore.ts`）：
- 使用 `defineStore` with options API style
- API 调用使用 `fetch` + `useUserStore()` 获取 token（参考现有 store 的 auth header 写法）
- state 中的通知列表类型定义需与后端响应字段一一对应

### 通知触发场景（本 Story 提供基础设施，其他 Story 触发）

- 额度用完提醒（Story 3.4）

# Story 4.1: 新手引导 3 步互动卡片

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

作为新注册用户，
我希望首次进入平台时体验 3 步互动引导卡片，
以便快速了解 QYQuant 的核心功能，不被复杂界面劝退。

## Acceptance Criteria

1. **Given** 用户首次登录（`users.onboarding_completed = false`）
   **When** 进入平台首页
   **Then** 自动展示 3 步引导卡片：第 1 步"认识策略广场"、第 2 步"一键回测策略"、第 3 步"查看回测报告"
   **And** 每步卡片有一个主操作按钮，点击后高亮对应界面区域
   **And** 用户可随时点击"跳过引导"关闭，并写入 `users.onboarding_completed = true`
   **And** 引导完成后不再重复展示（FR7）

## Dev Notes

### 架构约束与模式

- **前端框架**: Vue 3 + Composition API + Pinia 状态管理
- **后端框架**: Flask + 蓝图组织，所有路由前缀 `/api/v1/`
- **数据库**: PostgreSQL，使用 Alembic 管理 migration
- **认证**: JWT Access Token（内存）+ Refresh Token（HttpOnly Cookie），通过 `GET /api/v1/auth/profile` 获取用户信息
- **时区规范**: 数据库存 UTC（TIMESTAMP WITH TIME ZONE），API 输出 ISO 8601 带 +08:00，禁止 `datetime.now()`

### UX 设计要求

- **渐进式信息披露**（UX-3）：不一次性暴露所有功能，3 步卡片逐步引导
- **极简金融仪表盘风格**（NFR26/UX-5）：白色主背景、红色强调色（仅 CTA）、卡片式布局
- **核心功能到达路径 ≤3 次点击**（UX-6）
- **设计令牌**:
  - 卡片圆角: 12px
  - 内边距: 20px
  - 间距: 16px
  - 标题字号: 20px / 600wt
  - 正文字号: 16px / 400wt
  - CTA 按钮色: #1A1A1A（黑色）
  - 关闭/跳过色: #94A3B8（灰色）

### 前端组件规范

- **组件路径**: `frontend/src/components/onboarding/OnboardingGuide.vue`
- **高亮实现**: 使用全屏 overlay（`position: fixed; z-index: 1000`）+ CSS `clip-path` 或 `box-shadow` 镂空目标区域
- **状态管理**: Pinia `useUserStore()` 中读取 `onboarding_completed`，完成后更新 store
- **性能目标**: 引导加载 <500ms

### 后端 API 规范

- **响应格式统一**:
  ```json
  { "data": { "onboarding_completed": true }, "meta": {} }
  ```
- **错误格式**:
  ```json
  { "error": { "code": "UNAUTHORIZED", "message": "...", "details": {} } }
  ```
- **蓝图**: 放在 `auth` 蓝图（profile 相关）或 `users` 蓝图下

### 无障碍要求（WCAG 2.1 AA）

- 文本对比度 ≥4.5:1（#1E293B on #FFFFFF）
- 交互元素最小尺寸 44×44px
- 完整 Tab 键导航支持
- Focus 样式: 2px 红色边框 + 2px offset
- 所有交互元素添加 `aria-label`

### 关键依赖

- **前置依赖**: Epic 2（用户注册/登录）必须完成，用户认证系统可用
- **Epic 4 内依赖**: Story 4.2（首次回测引导）依赖本 story 的引导卡片框架
- **无外部服务依赖**: 纯前端+后端实现，无需第三方库

### 防踩坑提醒

- **不要** 使用 localStorage 存储引导状态 — 必须存数据库 `users.onboarding_completed`，确保跨设备一致
- **不要** 在引导组件中硬编码页面布局信息 — 使用 `ref` 或 `data-onboarding-target` 属性定位高亮区域
- **不要** 创建新的 HTTP 客户端 — 复用项目已有的 axios 实例（含 JWT 拦截器）
- **不要** 自行实现 overlay 库 — 如项目已有 UI 框架（如 Element Plus/Naive UI），优先使用其 overlay/popover 能力

### Project Structure Notes

- 前端组件放 `frontend/src/components/onboarding/` 目录
- 后端 API 放对应蓝图目录下
- Migration 文件遵循 Alembic 命名规范
- 测试文件与源码目录结构镜像

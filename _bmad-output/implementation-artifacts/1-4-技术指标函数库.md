## Story

作为策略开发者，
我希望直接使用预置的技术指标函数（SMA、EMA、ATR、布林带、金叉/死叉检测），
以便专注于策略逻辑而无需重复实现常用计算。

## Acceptance Criteria

1. `sma(series, period=20)` 接受 pandas Series 输入，返回等长的 pandas Series，数值与 `pandas.rolling(period).mean()` 一致，前 `period-1` 个值为 NaN
2. `ema(series, period=20)` 返回指数移动平均 Series，数值与 `pandas.ewm(span=period, adjust=False).mean()` 一致
3. `atr(high, low, close, period=14)` 接受三个 pandas Series，返回平均真实波幅 Series
4. `bollinger_bands(series, period=20, num_std=2.0)` 返回三元组 `(upper, middle, lower)` 三个 pandas Series
5. `cross_over(s1, s2)` 返回布尔 Series，在 s1 上穿 s2 的 K 线处为 True（前一根 s1 <= s2 且当前 s1 > s2）
6. `cross_under(s1, s2)` 返回布尔 Series，在 s1 下穿 s2 的 K 线处为 True（前一根 s1 >= s2 且当前 s1 < s2）
7. `tests/test_indicators.py` 单元测试分支覆盖率 ≥ 80%

## Dev Notes

### 核心设计决策

**1. pandas 作为运行时依赖而非 SDK 硬依赖**

架构决策要求 SDK "零依赖"（指不依赖 ta-lib 等 C 扩展），但 indicators 函数接受并返回 pandas Series。pandas 在策略执行环境（E2B 沙箱）中始终可用，因此：
- `indicators.py` 直接 `import pandas as pd`（运行时需要）
- **不在 `pyproject.toml` 的 `dependencies` 中添加 pandas**（保持 SDK 包轻量）
- 在 pyproject.toml 中添加 `pandas` 为 optional dependency（如 `[project.optional-dependencies] indicators = ["pandas>=2.0"]`）
- 类型注解使用 `pandas.Series`，配合 `TYPE_CHECKING` 保护

**关键约定：** 如果用户环境没有 pandas，导入 `indicators` 模块时会抛出 `ImportError`，这是预期行为——指标函数本身就需要 pandas 数据结构。

**2. 实现策略 — 直接使用 pandas 内置方法**

所有指标函数基于 pandas 内置的 rolling/ewm 方法实现，理由：
- pandas `rolling().mean()` 已是工业标准实现，无需重复造轮
- 性能足够（策略级数据量通常 < 10万根 K 线）
- 确保数值精度与用户手动计算一致
- 代码简洁可维护

**不使用 ta-lib 的理由：** [Source: architecture.md#决策 2: SDK 依赖策略]
- ta-lib 依赖 C 库，Windows 安装常失败
- 架构明确选择"零依赖 + 核心函数自实现"

**3. 函数式 API 设计**

指标函数采用纯函数式设计（无类、无状态），API 签名与业界标准对齐：
```python
def sma(series: pd.Series, period: int = 20) -> pd.Series: ...
def ema(series: pd.Series, period: int = 20) -> pd.Series: ...
def atr(high: pd.Series, low: pd.Series, close: pd.Series, period: int = 14) -> pd.Series: ...
def bollinger_bands(series: pd.Series, period: int = 20, num_std: float = 2.0) -> tuple[pd.Series, pd.Series, pd.Series]: ...
def cross_over(s1: pd.Series, s2: pd.Series) -> pd.Series: ...
def cross_under(s1: pd.Series, s2: pd.Series) -> pd.Series: ...
```

**4. NaN 处理策略**

- SMA/ATR 前 `period-1` 个值为 NaN（rolling 窗口不足）
- EMA 从第一个值开始就有结果（ewm 特性）
- cross_over/cross_under 第一个元素为 False（无前一根数据）
- 这与 pandas 原生行为一致，策略开发者已熟悉

**5. 参数验证规则**

- `period` 必须为正整数（> 0），否则抛出 `ValueError`
- `series` 不能为空（长度 = 0），否则抛出 `ValueError`
- `num_std` 必须为正数（> 0），否则抛出 `ValueError`
- ATR 的三个 Series 长度必须一致，否则抛出 `ValueError`

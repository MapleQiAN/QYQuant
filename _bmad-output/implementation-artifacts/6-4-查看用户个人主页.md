# Story 6.4: 查看用户个人主页

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

作为用户，
我希望查看其他用户的个人主页，了解其发布的策略和帖子，
以便发现优秀的策略开发者。

## Acceptance Criteria

1. **Given** 用户点击帖子或策略卡片上的作者头像/昵称
   **When** 访问 `GET /api/v1/users/:id`
   **Then** 展示该用户公开信息：头像、昵称、简介、注册时间（FR39）
   **And** 不展示手机号等敏感信息
   **And** 已软删除用户（`deleted_at` 非空）返回 `404`

2. **Given** 用户访问某用户个人主页
   **When** 调用 `GET /api/v1/users/:id/strategies`
   **Then** 返回该用户在广场公开的策略列表（`is_public=true`），按 `created_at` 降序排列
   **And** 支持分页（默认每页 20 条）
   **And** 每条策略含：id、name、category、returns、max_drawdown、win_rate、tags
   **And** 若该用户无公开策略，返回空列表

3. **Given** 用户访问某用户个人主页
   **When** 调用 `GET /api/v1/users/:id/posts`
   **Then** 返回该用户发布的帖子列表（社区帖，来自新 `posts` 表），按 `created_at` 降序排列
   **And** 支持分页（默认每页 20 条）
   **And** 每条帖子含：id、content（截断至 200 字）、likes_count、comments_count、created_at（北京时间）
   **And** 若该用户无帖子，返回空列表

4. **Given** 请求的 user_id 不存在或已软删除
   **When** 调用任意 `/api/v1/users/:id/*` 端点
   **Then** 返回 `404` 错误

5. **Given** 所有用户主页相关端点
   **When** 未认证或已认证用户调用
   **Then** 三个端点均无需认证（公开数据）

    ```python
    @bp.get('/<user_id>/posts')
    def get_user_posts(user_id):
        user, error = _get_user_or_404(user_id)
        if error:
            return error

        page = request.args.get('page', 1, type=int)
        per_page = min(request.args.get('per_page', 20, type=int), 50)

        pagination = (
            Post.query
            .filter(Post.user_id == user_id, Post.content.isnot(None))
            .order_by(Post.created_at.desc())
            .paginate(page=page, per_page=per_page, error_out=False)
        )

        items = []
        for p in pagination.items:
            content = p.content or ''
            items.append({
                'id': p.id,
                'content': content[:200],         # 截断至 200 字
                'likes_count': p.likes_count or 0,
                'comments_count': p.comments_count or 0,
                'created_at': format_beijing_iso(p.created_at) if p.created_at else None,
            })

        return ok({
            'items': items,
            'total': pagination.total,
            'page': page,
            'per_page': per_page,
        })
    ```

## Dev Notes

### 架构约束与模式

- **前端框架**: Vue 3 + Composition API + TypeScript + Pinia
- **后端框架**: Flask + flask-smorest 蓝图，用 `@bp.get()` 装饰器定义路由
- **认证**: 三个端点均无需认证（公开数据），不加 `@jwt_required()`
- **响应格式**: `ok(data)` 封装（来自 `app.utils.response`），错误用 `error_response(code, message, status)`
- **时区**: 帖子的 `created_at`（来自 Story 6.1，TIMESTAMPTZ 类型）用 `format_beijing_iso()` 转北京时间；策略的 `created_at` 目前为 BigInteger 毫秒时间戳

### ⚠️ 关键陷阱：GET /api/v1/users/:id 已存在（Story 2.3）

**Story 2.3 已在 `backend/app/blueprints/users.py` 中实现 `GET /api/v1/users/:id`：**
```python
@bp.get('/<user_id>')
def get_user_profile(user_id):
    user, error = _get_user_or_404(user_id)
    if error:
        return error
    return ok(_public_schema.dump(user))
```

**Story 6.4 不需要修改此端点**，只需在同一蓝图中**追加两个新路由**：
- `GET /api/v1/users/<user_id>/strategies`
- `GET /api/v1/users/<user_id>/posts`

**绝对不要**修改或删除已有的 `get_user_profile` 函数，Story 2.3 的测试依赖它。

### ⚠️ 关键陷阱：路由顺序（Flask 路由匹配优先级）

Flask 路由 `/<user_id>` 是通配符路由，可能与 `/me` 冲突。现有代码中 `/me` 路由已在 `/<user_id>` 之前注册，所以没问题。

新增的 `/<user_id>/strategies` 和 `/<user_id>/posts` 不会与 `/me` 冲突，安全添加。

### ⚠️ 关键陷阱：is_public 字段依赖 Story 5-4

`strategies.is_public` 字段**尚不存在**于当前数据库，由 Story 5-4（策略发布-免费分享模式）的 Alembic 迁移添加。

**在 Story 5-4 未完成前，`GET /api/v1/users/:id/strategies` 会因 `is_public` 列不存在而报 SQLAlchemy 错误。**

**处理方案**：
```python
# ✅ 临时兼容方案（若 Story 5-4 尚未完成）：
# 在 Strategy 模型中临时用 status 模拟（不推荐，仅临时）
.filter_by(owner_id=user_id)
.filter(Strategy.status == 'published')  # 暂替代 is_public

# ✅ 正式方案（Story 5-4 完成后）：
.filter_by(owner_id=user_id, is_public=True)
```

本 Story 实现时，若 Story 5-4 已完成，直接使用 `is_public=True`；若尚未完成，使用 `status == 'published'` 临时替代，并在 Story 5-4 完成后更新。

### ⚠️ 关键陷阱：Post 新字段依赖 Story 6.1

`Post.content`、`Post.likes_count`、`Post.comments_count`、`Post.created_at`（TIMESTAMPTZ）字段由 Story 6.1 的 Alembic 迁移添加（`posts` 表现有字段为 `title/author/avatar/likes/comments/timestamp`）。

`Post.content.isnot(None)` 过滤只返回新版帖子（有 content 的），旧版帖子（forum.py 创建，content 为 NULL）不出现在用户主页。

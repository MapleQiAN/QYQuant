# Story 4.4: 全站分层合规免责声明

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

作为平台，
我希望在全站各关键场景展示分层的合规免责声明，
以便满足监管要求并管理用户预期。

## Acceptance Criteria

1. **Given** 新用户注册时
   **When** 提交注册表单
   **Then** 必须勾选服务协议（含免责条款）才能提交，否则提交按钮置灰（NFR22，FR55）

2. **Given** 用户查看回测结果页
   **When** 页面加载完成
   **Then** 底部显示灰色小字"基于历史数据，不构成投资建议"
   **And** 免责文本在页面 DOM 中始终存在，不因任何用户操作消失

3. **Given** 用户查看策略详情页的收益数据
   **When** 悬停在收益数字上
   **Then** tooltip 显示"仅供参考，历史表现不代表未来收益"
   **And** 上述免责文本在页面 DOM 中始终存在，不因任何用户操作消失

## Dev Notes

### 架构约束与模式

- **前端框架**: Vue 3 + Composition API + `<script setup>` + TypeScript
- **状态管理**: Pinia stores（全局状态）；组件级状态用 `ref()`
- **CSS**: 自定义 CSS 设计令牌体系（无 UI 组件库），CSS 变量定义在 `global.css`
- **组件命名**: PascalCase.vue（`DisclaimerFooter.vue`、`DisclaimerTooltip.vue`）
- **测试**: 组件同目录 co-located（`DisclaimerFooter.vue` + `DisclaimerFooter.test.ts`）
- **图标**: inline SVG + `h()` 渲染（沿用 TopNav.vue BellIcon 模式），不引入图标库
- **无 i18n**: 免责声明文本直接用中文硬编码在 `disclaimer-content.ts` 中

### 本 Story 的技术特点

本 story 是**纯前端实现**，无需新增后端 API 或数据库变更：
- 免责声明文本为静态常量，存储在 `frontend/src/data/disclaimer-content.ts`
- 不依赖用户登录状态（免责声明对所有用户可见）
- SimulationDisclaimerModal 的"已知晓"状态 MVP 阶段使用 `localStorage` 暂存，待 Epic 7 模拟托管实现时迁移至后端 `users.sim_disclaimer_accepted` 字段

### UX 设计要求

- **免责声明设计原则**（PRD + UX 规范）：
  - 免责声明用灰色小字（12px，`#94A3B8`），不打扰主流程
  - 回测报告底部常驻小字（不可隐藏、不可消失）
  - 模拟托管首次进入一次性确认，此后不再弹出
  - **禁止**: 每次打开页面都强制弹出免责声明（UX 反模式）
- **字体规范**: 免责声明专用 12px / 400 weight（UX 设计系统 Typography 小字层级）
- **弹窗模式表**:
  - Modal 弹窗：需要用户决策，有遮罩，必须有明确关闭按钮，点击外部不关闭
  - Tooltip：hover 触发，300ms 延迟
- **颜色系统**:
  - 免责文本色: `#94A3B8`（辅助文本/小字）
  - 分割线: `#E5E5E5`
  - CTA 按钮: `#1A1A1A`
  - 链接色: `#3B82F6`
  - 次要文本: `#64748B`

### 防踩坑提醒

- **不要** 使用 `v-if` 控制免责声明显示/隐藏 — 回测页和策略详情页的免责文本必须始终在 DOM 中渲染
- **不要** 每次打开页面都弹出免责声明弹窗 — 模拟托管免责仅首次进入弹出，使用 localStorage 记录
- **不要** 引入第三方 Tooltip 库 — 沿用 Story 4.3 的自实现 tooltip 模式
- **不要** 引入第三方 Modal 库 — 自行实现轻量 Modal，使用 `<Teleport to="body">`
- **不要** 将免责声明文本硬编码在组件内部 — 统一维护在 `disclaimer-content.ts`
- **不要** 在 DisclaimerTooltip 中复用 Story 4.3 的 MetricTooltip — 两者职责不同（指标解释 vs 免责提示），可参考其实现模式但保持独立
- **不要** 创建后端 API — 本 story 纯前端实现，模拟托管已知晓状态暂存 localStorage

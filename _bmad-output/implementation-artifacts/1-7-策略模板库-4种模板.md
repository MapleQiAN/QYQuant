# Story 1.7: 策略模板库（4 种模板）

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

作为策略开发者，
我希望通过 `qys init --template` 从 4 种内置模板快速创建新策略，
以便从一个可运行、有注释的示例开始，无需从零编写框架代码。

## Acceptance Criteria

1. **Given** 4 种模板：trend-following、mean-reversion、momentum、multi-indicator，**When** 运行 `qys init my-strat --template mean-reversion`，**Then** 创建的策略直接通过 `qys validate`，无任何报错
2. 每个模板使用 event_v1 接口（`on_bar` 带类型注解的 `ctx: StrategyContext` 和 `data: BarData` 参数）和 SDK 数据类
3. 每个模板通过 `ctx.parameters` 读取至少一个参数，并设有文档化的默认值
4. 行内注释解释关键逻辑（入场/出场条件、参数作用）
5. 每个模板均包含对应的 `strategy.json`（含合理的 parameters 定义）和 `src/strategy.py`（有实际策略逻辑）和 `README.md`
6. `qys init` 命令根据 `--template` 选项从 `packages/qysp/src/qysp/templates/` 目录加载模板文件，替代当前硬编码最小模板
7. 所有命令在 Windows 10+、macOS 12+、Ubuntu 20.04+ 上正常工作
8. `tests/test_cli.py` 中新增模板相关测试，整体分支覆盖率 ≥ 80%

## Dev Notes

### 核心设计决策

**1. 模板存储方式 — 文件系统模板（非字符串内嵌）**

Story 1.6 使用硬编码字符串作为最小模板。Story 1.7 改为文件系统模板：每个模板类型在 `templates/` 下有独立子目录，包含真实的 `strategy.json`、`strategy.py`、`README.md` 文件。

优势：
- 模板文件可独立编辑和测试
- `qys init` 只需复制文件 + 替换少量字段
- 未来添加新模板只需增加子目录

文件加载使用 `importlib.resources`（与 Story 1.5 schema 加载方式一致），确保 pip install 后模板文件正确打包。

[Source: architecture.md#起点模板 — 项目结构 templates/]
[Source: epics.md#Story 1.7 验收标准]

**2. 模板目录命名 — 下划线风格**

模板目录使用 `snake_case`（如 `trend_following/`），与 Python 包命名一致。CLI 参数仍使用 `kebab-case`（如 `--template trend-following`），在加载时将 `-` 转为 `_`。

**3. 4 种策略模板设计**

| 模板 | 核心指标 | 策略逻辑 | 关键参数 |
|------|---------|---------|---------|
| trend-following | SMA（双均线） | 短期均线上穿长期均线买入，下穿卖出 | `fast_period`=5, `slow_period`=20 |
| mean-reversion | Bollinger Bands | 价格跌破下轨买入，涨破上轨卖出 | `bb_period`=20, `bb_std`=2.0 |
| momentum | EMA + cross_over | EMA 动量确认，EMA 方向一致时入场 | `ema_period`=12, `signal_period`=26 |
| multi-indicator | SMA + ATR + Bollinger | 多信号综合判断，多数信号一致才触发 | `sma_period`=20, `atr_period`=14, `bb_period`=20 |

每个模板使用 `qysp.indicators` 中已有的函数（sma、ema、atr、bollinger_bands、cross_over、cross_under），不引入新依赖。

[Source: prd-qystrategy-improvement.md#FR-6 策略模板]
[Source: indicators.py — sma/ema/atr/bollinger_bands/cross_over/cross_under]

**4. init 命令重构 — 最小改动原则**

只修改 `init` 函数体中的模板生成逻辑，不改变命令签名、参数验证、目录创建等外围代码。具体变化：
- 删除 `_STRATEGY_PY_TEMPLATE` 常量
- 删除 `_make_strategy_json` 函数
- `init` 函数内调用 `templates.load_template_files(template)` 获取内容
- 动态替换 `strategy.json` 中的 `name` 和 `id` 字段

**5. strategy.py 中使用 SDK 类型注解**

每个模板的 `strategy.py` 顶部导入 SDK 数据类：
```python
from __future__ import annotations
from qysp.context import StrategyContext, BarData
```

`on_bar` 签名：
```python
def on_bar(ctx: StrategyContext, data: BarData) -> list:
```

注意：模板代码中使用 `ctx.parameters.get("key", default)` 读取参数，不硬编码参数值。

[Source: context.py — StrategyContext, BarData, OnBarCallable 定义]

### 前一个 Story 经验（Story 1.6）

**关键学习与注意事项：**

1. **pytest-cov + Windows** — `--cov=qysp.xxx` 在 Windows 可能因模块多次加载报错，改用 `--cov=qysp` 全包覆盖可绕过
2. **测试运行格式** — 使用 `uv run --directory packages/qysp pytest tests/` 格式
3. **分支覆盖率 ≥ 80%** — 使用 `--cov-branch` 检查

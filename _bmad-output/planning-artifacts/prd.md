# Product Requirements Document - QYQuant

**Author:** Serendy
**Date:** 2026-03-10

---

## 执行摘要

**QYQuant** 是一个面向所有投资者的量化交易平台，核心定位：**让完全不懂编程、不懂量化的人，也能使用量化策略投资。**

**核心差异化**：策略即服务（Strategy-as-a-Service）——策略如 App Store 应用，用户一键选用、一键回测、一键模拟托管，无需写任何代码。现有平台（聚宽、米筐、掘金）均要求用户具备 Python 编程能力，QYQuant 填补"零编程门槛 + 策略市场 + 社区"的完整闭环空白。

**目标用户**：量化小白散户（核心）、量化爱好者/程序员（进阶）、策略开发者/卖家（供给侧）。

**MVP 三模块**：量化回测引擎（核心）+ 策略广场（展示与社区）+ 模拟托管（过渡版，不涉及真实资金）。通过免费 100 次/月回测额度获客，付费订阅（200/500/1000 元/月）变现。

**关键风险**：聚宽数据商用授权为阻断性问题，需在 MVP 上线前解决；MVP 阶段规避策略付费售卖和真实资金托管以降低合规风险。

## 成功标准

### 用户成功

**量化回测模块**
- 用户从上传策略代码到获得完整回测报告，全流程在 **30 秒内**完成
- 回测报告默认重点展示**前 3 项核心指标**：累计收益率、最大回撤、夏普比率（其余指标折叠展示）
- 用户首次回测成功率 > 80%（非平台问题导致的失败）

**模拟托管模块（MVP 过渡版）**
- 用户设置模拟托管后，策略连续稳定运行 **30 天无中断**
- 用户能清晰看到模拟持仓、模拟收益曲线、买卖信号记录

**策略广场（展示层）**
- 用户浏览到感兴趣的策略后，**2 步以内**完成"一键导入回测"
- 策略展示页让用户在不看代码的前提下判断策略是否值得试用

### 业务成功

| 阶段 | 指标 | 目标值 | 基准参考 |
|------|------|--------|----------|
| 上线 3 个月 | 注册用户数 | 1,000+ | 冷启动验证 |
| 上线 3 个月 | 免费→付费转化率 | ≥ 3% | SaaS 行业均值 2-5% |
| 上线 6 个月 | 免费→付费转化率 | ≥ 5% | 优质 SaaS 平均水平 |
| 上线 6 个月 | 月活用户留存率（Day 30）| ≥ 25% | 工具类 SaaS 健康值 |
| 上线 6 个月 | 策略广场策略数量 | 50+ 条公开策略 | 内容生态冷启动门槛 |
| 上线 12 个月 | MRR（月经常性收入）| 正向增长趋势 | 验证商业可持续性 |

### 技术成功

- 回测任务执行成功率 ≥ 99%（非用户代码错误导致的失败）
- 系统可用性 ≥ 99.5%（月度统计）
- 数据源接入稳定，历史数据获取失败率 < 0.5%
- 策略代码与执行结果完全隔离，不对外暴露任何策略逻辑细节

### 可衡量结果

- **核心漏斗**：注册 → 首次回测 → 回测超过 3 次 → 付费转化
- **关键行为触发付费**：免费 100 次回测额度耗尽，触发充值/订阅转化
- **策略广场冷启动**：上线前通过内部/邀请渠道预备 20+ 条种子策略

## 产品范围

### MVP —— 最小可行产品（第一版上线）

**模块一：量化回测（核心）**
- 策略代码上传（Python）
- 对接聚宽数据源，获取历史行情数据
- 设定回测时间区间，执行回测
- 输出：买卖点标注、11 项量化评价指标
- 免费额度：100 次/月；超额付费解锁

**模块二：策略广场（展示层）**
- 用户发布策略（含基本指标展示，不公开代码）
- 策略一键导入回测（试用）
- 基础社区功能：发帖、点赞、评论、收藏
- 个人主页：头像、昵称、简介

**模块三：模拟托管（过渡版）**
- 用户选择策略 + 设置模拟资金
- 模拟机器人基于历史/实时行情运行策略
- 展示：模拟持仓、模拟收益曲线、买卖信号
- **不涉及真实资金，规避合规风险**

**订阅层级**

| 套餐 | 月费 | 回测次数 | 模拟托管槽位 |
|------|------|---------|-------------|
| 免费版 | 0 | 100 次/月 | 1 个 |
| 轻量版 | 200 元/月 | 200 次/月 | 2 个 |
| 进阶版 | 500 元/月 | 300 次/月 | 3 个 |
| 专业版 | 1000 元/月 | 800 次/月 | 5 个 |

### 成长功能（MVP 后）

- 真实券商 API 对接（合规路径明确后）
- 策略付费售卖（买断制、订阅制）
- 用户私聊、打赏、勋章体系
- 帖子转发至外部社交平台（微信、Telegram 等）
- AI 策略生成器（自然语言→策略代码）
- 学习中心（图文/视频课程 + "立即回测"按钮）
- 收益分享卡片（一键生成带水印截图）
- 销售数据看板（策略开发者）
- 备选数据源（Tushare Pro）
- 管理后台增强（财务报表/数据源监控告警）

### 愿景（未来）

- 真实资金托管 + 收益分成制（合规牌照申请后）
- 可视化策略搭建（模块化拖拽积木）
- 企业/机构账户（多账户管理、团队协作）
- 策略争议仲裁机制
- 策略智能推荐算法
- 国际化扩展（海外券商 API、多语言）

## 差异化战略与市场定位

### 核心定位

**"QYQuant —— 人人都能用的量化交易平台。"**

### 差异化优先级

1. **MVP — 策略即服务（Strategy-as-a-Service）**：策略如同 App Store 应用，小白用户一键选用、一键回测、一键模拟托管，无需写任何代码
2. **成长期 — AI 策略生成器**：用户用自然语言描述投资逻辑，平台 AI 自动生成策略代码并执行回测
3. **愿景期 — 可视化策略搭建**：模块化拖拽组合策略条件，如搭积木般构建个性化量化策略

### 市场背景与竞争格局

- 中国近 2 亿散户中，绝大多数无编程能力，被现有量化平台劝退
- BigQuant 做了可视化搭建，但仍需理解量化概念，不算"小白可用"
- 聚宽有策略商城，但生态弱、社区不活跃、策略保护机制不完善
- **空白地带**：没有一个平台真正做到"零编程门槛 + 策略市场 + 社区"的完整闭环

| 创新维度 | 现有市场 | QYQuant |
|---------|---------|---------|
| 目标用户 | 会写 Python 的量化工程师 | 所有投资者，包括完全不懂编程的人 |
| 策略获取方式 | 自己写代码 | 策略广场选用（如选 App）、AI 生成（自然语言）、自己写代码 |
| 核心体验 | 开发环境 + API 文档 | 一键回测 + 一键模拟托管 |
| 策略分发模式 | 代码开源分享 | 加密执行，使用权与代码分离（皮肤模式） |

### 四层创新路径（递进式）

1. **策略即服务**：策略广场策略如 App Store 应用，选用即运行（MVP）
2. **体验创新**：新手引导 + 学习路径 + 每个知识点以"动手操作"结尾（MVP 基础版）
3. **架构创新**：策略执行即服务（Strategy Execution as a Service），一石三鸟解决易用性、代码安全、知识产权（MVP）
4. **生态创新**：学习中心 + 策略广场 + 开发者变现形成内容飞轮（成长阶段）

### 学习功能分层

| 阶段 | 学习功能 | 复杂度 |
|------|---------|--------|
| **MVP** | 新手引导（3 步互动卡片 + 首次回测引导） | 低 |
| **MVP** | 帮助文档 / FAQ（什么是回测？什么是夏普比率？） | 低 |
| **成长期** | 学习中心（图文/短视频入门课程 + "立即回测"按钮） | 中 |
| **愿景期** | 社区贡献课程 + 策略开发者教程 + 课程付费 | 高 |

### 验证方法

- **MVP 核心假设**：不会编程的人，是否愿意使用他人的量化策略进行回测和模拟投资？
- **关键指标**：非程序员用户的注册→首次回测转化率（目标 > 50%）
- **验证周期**：上线后 3 个月内收集足够样本
- **失败信号**：非程序员用户注册后 7 天内未完成首次回测，说明"易用性"承诺未兑现

### 风险缓解

| 创新风险 | 缓解策略 |
|---------|---------|
| 小白用户可能不信任"别人的策略" | 平台验证徽章 + 免费回测试用，降低信任门槛 |
| 小白用户看不懂回测指标 | 新手引导 + FAQ + 指标 tooltip 解释 |
| AI 生成策略质量不可控 | 成长阶段上线，加"AI 生成仅供参考"免责 + 人工审核 |
| 策略加密可能被逆向破解 | Docker 沙箱执行 + 字节码编译，不存储明文代码 |
| "皮肤模式"用户接受度未知 | MVP 先做免费策略验证模式是否成立，再扩展到付费 |
| 用户误以为"回测收益 = 实盘收益" | 全站多层免责提示 + 新手引导明确教育"回测 ≠ 实盘" |

## 用户旅程

### 旅程一：量化小白老王 —— 零代码策略体验之旅

**开场**：老王 50 岁，有 50 万闲钱炒股，完全不会编程。他听说"量化交易"能赚钱，搜索后发现聚宽、米筐全要写 Python，直接劝退。后来朋友推荐了 QYQuant，说"不用写代码也能用"。

**行动**：注册账号（手机号一键注册）→ 进入策略广场，看到精选策略推荐位，第一个就是"沪深 300 均线趋势策略"，标注着 **"平台验证回测"** 的蓝色徽章，年化 18%，最大回撤 12% → 点击策略详情页，看到 Equity Curve 资金曲线图一眼就懂 → 点击"免费回测试用"，选标的、选时间段，一键运行 → 25 秒后报告出来，收益率、最大回撤、夏普比率三项核心指标醒目展示。

**高潮**：老王觉得这个策略还不错，点击"导入我的策略库" → 进入模拟托管，设置 50 万模拟资金 → 模拟机器人开始运行，每天打开平台看模拟持仓和收益曲线。两周后，他感受到"如果是真钱，我已经赚了 3000 块"。平台弹出提示："模拟结果仅供参考，实盘可能因滑点和流动性产生差异"。

**结局**：老王成为日活用户，免费回测额度快要用完。他升级了轻量版套餐（200 元/月）。他用平台的**收益分享卡片**功能，一键生成带 QYQuant 水印的收益截图，发到朋友圈——"量化交易原来这么简单！"

**揭示需求**：零代码策略使用路径、精选推荐机制、平台验证徽章、Equity Curve 图、模拟托管免责提示、收益分享卡片、套餐升级引导。

### 旅程二：量化爱好者小明 —— 代码回测进阶之旅

**开场**：小明 27 岁，IT 工程师，会 Python，写了一个均线交叉策略想验证效果。

**行动**：注册 → 进入量化回测模块 → 上传 Python 策略文件 → 平台提示支持的依赖库清单（pandas、numpy、ta-lib 等）→ 选择沪深 300 + 中证 500 组合回测 → 设定 2020-2024 区间 → 点击运行。

**高潮**：25 秒出结果。核心三指标醒目展示，Equity Curve 直观呈现策略稳定性。他展开完整 11 项指标，发现手续费占比偏高，于是调整持仓周期参数。

**失败兜底**：第二次上传时代码报错。平台不只是显示行号，而是弹出**智能错误提示**："第 23 行 `context.buy()` 缺少 `amount` 参数，[点击查看示例代码]"。他修改后重新运行成功。

**结局**：小明连跑 8 次回测，优化了参数。他把最终版策略发布到策略广场，设置为免费分享。一周后收到 15 个赞和 3 条评论。他的回测额度剩余 80 次（每次运行计 1 次，调参重跑也计 1 次，规则在运行按钮旁明确标注）。

**揭示需求**：代码上传、依赖库支持清单、多标的组合回测、智能错误提示、额度计算规则透明展示、策略发布到策略广场。

### 旅程三：策略开发者 Lisa —— 策略变现之旅

**开场**：Lisa 全职量化，手上有一套实盘跑了两年的策略，年化 23%，最大回撤 8%。她想在 QYQuant 上变现。

**行动**：进入策略广场 → "发布策略" → 填写名称、描述 → 上传策略文件 → 平台弹出**代码保护说明**："您的策略代码将在服务端沙箱环境中编译执行，买家仅获得运行权，无法查看、下载或导出源码。[查看知识产权保护协议]" → 选择展示指标 → 设定定价：买断制 2999 元。

**关键节点**：发布前，平台展示**退款与争议处理政策**："买家购买后 7 天内，若策略无法正常运行（非市场因素），可申请退款。平台将核实后处理。详见[争议处理规则]"。Lisa 看完后觉得公平合理，点击发布。

**高潮**：一周后收到第一笔 2999 元购买通知，平台扣除 30% 服务费后到账 2099 元。她在**销售数据看板**中看到：本月销量 3 笔、总收入 6298 元、策略浏览量 420 次、转化率 0.7%。

**结局**：Lisa 建立了个人主页，积累 200 关注者。她开始规划第二个策略的发布，并考虑增加"订阅制"定价选项。

**揭示需求**：代码沙箱保护机制说明、知识产权协议、退款争议处理政策、销售数据看板（销量/收入/浏览/转化率）、多定价模式。

### 旅程四：平台管理员小张 —— 运营管理之旅

**开场**：小张是 QYQuant 运营人员，每天需要管理内容、监控系统、处理异常。

**日常工作流**：
- **内容审核**：查看待审核策略列表 → 审查描述和指标合规性 → 通过/拒绝 → 处理用户举报（数据造假/违规内容）→ 下架违规策略并通知作者
- **系统监控**：查看回测任务队列状态（排队数、平均耗时、失败率）→ 手动干预卡死任务 → 数据源健康看板（聚宽 API 状态，异常自动告警邮件）
- **用户管理**：封禁违规用户、查看操作日志、重置密码
- **财务对账**：查看月度服务费收入、策略交易抽成、分成结算报表 → 导出 Excel

**揭示需求**：管理后台、策略审核队列、举报处理、任务队列监控与干预、数据源健康告警、用户管理、财务报表导出。

## 领域特定需求

### 合规与监管

**平台资质（MVP 阶段）：**
- ICP 许可证（增值电信业务经营许可证）—— MVP 上线前必须取得
- 全站免责声明："本平台不构成投资建议，策略回测结果仅供参考，不代表未来收益"
- MVP 不涉及付费策略售卖，仅支持免费策略分享，不触发金融信息服务备案要求

**平台资质（成长阶段，解锁策略售卖时）：**
- 金融信息服务备案
- 证券投资咨询资质评估
- 策略交易资金走第三方支付清算，不在平台沉淀

**程序化交易新规（2025年7月已实施）：**
- MVP 仅做模拟托管，不触发程序化交易报告义务
- 未来真实资金托管需与券商合作完成报告备案
- 高频交易认定标准：单账户每秒 ≥ 300 笔或全日 ≥ 20000 笔

### 数据源合规

- 聚宽数据商用授权 —— **阻断性问题**，MVP 上线前必须签订商用协议
- 双线并行：联系聚宽商务 + 评估 Tushare Pro 作为备选数据源
- 本地缓存历史数据以降低 API 调用量（需确认缓存是否在授权范围内）

### 数据安全与隐私

- 策略代码保护：Docker 容器隔离执行 + AES-256 加密存储，执行完毕后容器销毁（详见 NFR10、NFR11）
- 用户协议明确："平台不主动访问、不泄露用户策略代码"
- 回测数据使用权：用户回测结果归用户所有，平台仅有权使用匿名化聚合数据
- 数据分类分级：策略代码（高敏感）、回测结果（中敏感）、帖子内容（低敏感）
- 隐私政策集中公开展示 + 用户知情同意机制 + 数据可删除（被遗忘权）

## 项目范围与平台架构

### MVP 策略与哲学

**MVP 类型：体验驱动型 + 平台搭建型混合**

核心原则：三模块基础闭环 + 新手引导体验做到极致。用户第一次打开 QYQuant 到完成首次回测的体验，决定一切。

### MVP 功能集

**核心用户旅程支撑：**
- 旅程一（小白老王）：零代码选策略 → 一键回测 → 模拟托管 — 完整支撑
- 旅程二（爱好者小明）：上传代码 → 回测 → 发布到策略广场 — 完整支撑
- 旅程四（管理员小张）：内容审核 + 系统监控 — 基础版支撑

**Must-Have 清单：**

| 模块 | 功能 | 不做会怎样？ |
|------|------|-------------|
| 用户系统 | 注册/登录（手机号）、个人主页 | 产品无法使用 |
| 新手引导 | 3 步互动卡片 + 首次回测引导 | 核心假设无法验证，小白用户流失 |
| 量化回测引擎 | 策略上传/选用 → 设定参数 → 执行 → 报告 | 产品不存在 |
| 回测报告 | 核心三指标 + Equity Curve + 11 项指标折叠 + 买卖点标注 | 回测价值大打折扣 |
| 智能错误提示 | 代码报错时给出可读提示和示例引导 | 程序员用户首次失败即流失 |
| 策略广场 | 策略列表 + 详情页 + 平台验证徽章 + 免费分享发布 | 小白用户无策略可用 |
| 精选机制 | 人工运营精选策略推荐位 | 新用户进入策略广场看不到优质内容 |
| 一键导入回测 | 策略广场策略 → 导入策略库 → 一键回测 | 策略即服务体验断裂 |
| 模拟托管 | 选策略 + 设模拟资金 → 模拟机器人运行 → 持仓/收益展示 | 缺少持续使用动力 |
| 社区基础功能 | 发帖、点赞、评论、收藏 | 策略广场无互动，内容生态冷启动失败 |
| 订阅套餐 | 4 级套餐 + 微信支付/支付宝 | 无法变现 |
| 额度管理 | 回测次数统计 + 套餐升级引导 + 规则透明展示 | 用户对付费无感知 |
| 免责声明 | 分层设计（注册/回测/详情/模拟托管各场景） | 合规风险 |
| 帮助文档/FAQ | 基础量化概念解释 | 小白用户看不懂指标 |
| 管理后台（基础版） | 策略审核 + 用户管理 + 回测任务监控 | 运营无法开展 |
| 策略沙箱 | Docker 容器隔离执行 + 代码加密存储 | 代码安全无保障 |
| 数据缓存层 | 历史行情数据本地缓存 | API 调用爆量，系统不可用 |

### 技术架构考量

- **前端**：Web 应用（SPA），响应式设计，支持桌面和移动端浏览器
- **后端**：API 服务 + 量化回测执行引擎 + 策略沙箱
- **量化回测引擎**：Docker 容器隔离执行，每次任务独立容器，执行后销毁
- **数据缓存层**：本地缓存已请求的历史数据，降低 API 调用量
- **存储**：策略代码 AES-256 加密存储，解密密钥与代码分离
- **实时性**：模拟托管需要准实时行情推送（WebSocket）

- 浏览器兼容性：Chrome、Edge、Firefox、Safari 最新两个版本

### 风险缓解策略

**技术风险：**
- 最具挑战性：量化回测引擎 + Docker 沙箱执行的性能和稳定性
- 缓解：MVP 阶段限制并发回测数量（如最多 10 个同时执行），逐步扩容
- 最大不确定性：聚宽数据授权——如果谈不拢，需要在 MVP 前切换到 Tushare Pro

**市场风险：**
- 核心假设未验证："小白用户是否真的愿意使用他人策略"
- 缓解：MVP 上线前通过内部测试 + 20 名种子用户验证
- 失败信号：种子用户 7 天内首次回测完成率 < 50%

**资源风险：**
- 最小团队配置：1 全栈开发 + 1 后端开发（量化回测引擎）+ 1 产品/运营
- 如果资源更少：砍掉模拟托管，只做量化回测 + 策略广场两个模块
- 绝对最小集：量化回测引擎 + 策略广场（免费分享 + 一键导入回测）

## 非功能需求 (NFRs)

### 性能

- **NFR1**: 回测任务提交后，系统应在5秒内开始执行或进入排队状态并返回确认
- **NFR2**: 简单策略（单标的、1年数据、日线）回测应在30秒内完成；当历史数据已缓存时，应在10秒内完成
- **NFR3**: 复杂策略（多标的、5年数据、分钟线）回测应在5分钟内完成
- **NFR4**: 策略广场页面首屏加载时间应≤2秒（含策略卡片列表）
- **NFR5**: K线图表交互（缩放、拖拽、切换周期）响应时间应≤200ms

### 可用性与可靠性

- **NFR6**: 量化回测服务月度可用性应≥99.5%（不含计划维护窗口）
- **NFR7**: 数据缓存层应支持JoinQuant API不可用时仍可使用已缓存的历史数据回测；缓存命中时≥90%回测任务无需调用外部API，缓存未命中时降级仍可返回已缓存数据并展示明确的数据范围提示
- **NFR8**: 单个回测任务失败不应影响其他用户的回测任务（容器隔离）
- **NFR9**: 系统应支持优雅降级：当外部数据源不可用时，应在≤3秒内展示明确错误提示（含数据源状态说明），不出现空白页面或未捕获崩溃

### 安全性

- **NFR10**: 用户上传的策略代码必须在隔离的Docker容器沙箱中执行，禁止网络访问和文件系统写入
- **NFR11**: 存储的策略代码必须使用AES-256加密，解密密钥与密文分离存储
- **NFR12**: 用户密码必须使用bcrypt（cost factor ≥ 12）哈希存储
- **NFR13**: 所有API端点必须实施速率限制（免费用户：60次/分钟，付费用户：120次/分钟）
- **NFR14**: 敏感操作（支付、密码修改、绑定券商API）必须要求二次验证

### 可扩展性

- **NFR15**: 系统架构应支持从单机部署平滑迁移到分布式部署（微服务就绪）；各功能模块（回测引擎/策略广场/模拟托管）通过接口边界解耦，每个模块可独立启动并通过接口联通验证
- **NFR16**: 回测任务队列应支持水平扩展（添加更多Worker节点）；系统应能在15分钟内完成Worker节点扩容至当前数量2倍，扩容期间已排队任务不丢失
- **NFR17**: 模拟托管服务可用性应≥99.5%（初期阶段合理目标，随成熟度提升至99.9%）

### 用户体验

- **NFR18**: 新用户从注册到完成首次回测的操作步骤应≤5步
- **NFR19**: 所有核心功能页面应适配主流桌面浏览器（Chrome、Firefox、Edge最新两个版本）
- **NFR20**: 回测结果页面必须包含可交互Equity Curve图表（支持缩放、悬停显示数值），不得仅展示数字表格；图表应作为页面首屏默认可见内容
- **NFR21**: 错误提示应包含具体原因和建议操作，而非通用错误码

### 合规性

- **NFR22**: 平台所有页面应在适当位置展示投资风险免责声明
- **NFR23**: 用户数据存储和处理应符合《个人信息保护法》要求（数据最小化、明确授权、可删除）

### 小白友好性

- **NFR24**: 平台提供的预置策略首次回测成功率应≥95%（因平台原因导致的失败率≤5%）
- **NFR25**: 所有耗时超过3秒的操作必须提供进度反馈（进度条、百分比或阶段提示）

### UI设计风格

- **NFR26**: 全平台采用极简金融仪表盘风格：白色主背景、卡片式布局、扁平化设计。量化回测和模拟托管页面遵循已有设计稿方向；策略广场页面参考币安广场的内容卡片流布局，但保持白色主题一致性。所有页面需确保视觉复杂度低于传统量化平台，符合零基础用户的认知负荷要求。单屏可见数据指标不超过7个；任何核心功能到达路径不超过3次点击。颜色规范：CTA主操作按钮使用纯黑色（`#1A1A1A`）；红色（`#E53935`）仅用于涨跌指示和强调色，不用于CTA按钮。

### 数据一致性与运行环境

- **NFR27**: 回测结果可复现性——同一策略、同一时间段、同一参数，多次回测结果必须一致
- **NFR28**: 回测容器预热池——系统应维持预热容器池，容器冷启动时间≤1秒，确保回测任务可即时分配执行环境

### 审计与合规追踪

- **NFR29**: 平台应记录关键操作审计日志，覆盖范围：支付操作、管理员账户操作（用户封禁/策略下架/权限变更）、策略代码访问记录；日志保留期限≥90天，仅管理员角色可访问审计日志，日志数据不可篡改
